# 환자 관리 시스템 (PMS) — Experience Spine

> 동작(behavior) 계약서. 정보 구조·상태·인터랙션·접근성·여정을 소유한다.
> 시각 정체성(색·타이포·간격·elevation·shape)은 `DESIGN.md`에 있다 — 여기서는 토큰을 `{path.to.token}` 구문으로 **이름으로만** 참조하고, 시각 스펙을 재서술하지 않는다.
> 이 제품은 **shadcn/ui on Tailwind 4**를 상속한다. 따라서 이 문서는 shadcn 기본 동작 위의 **델타(delta)만** 명시한다. 명시 없는 동작은 shadcn 기본을 따른다.

## Foundation

**2개 서피스 (architecture 확정):**

| 서피스 | 폼팩터 | 기술 | 누가 |
|---|---|---|---|
| 직원 데스크톱 웹 | 데스크톱(윈도우 우선, Edge 기본 / Chrome) | Next.js 16 App Router · shadcn/ui on Tailwind 4 | 직원 5역할 |
| 환자 포털 | 반응형 웹(모바일) — Flutter `webview_flutter` 얇은 셸이 동일 Next 앱의 `(patient)` 포털을 로드 | 같은 Next 앱, 반응형 | 환자 |

직원 웹은 App Router **route group 5종** — `reception` · `doctor` · `nurse` · `radiology` · `admin` + `(auth)/login`. 환자는 `(patient)/booking` · `(patient)/records`. `(staff)`·`(patient)` = "청중 경계". 환자 서피스는 네이티브 UI가 아니라 **반응형 웹**이며, Flutter는 푸시·생체인증(시뮬) 확장 여지를 가진 얇은 셸일 뿐이다. 접수 태블릿 서피스는 없다(데스크톱 Chromium 전용).

**UI 시스템 = shadcn/ui (own-the-components).** 그리드=TanStack Table, 폼=React Hook Form 7 + Zod 4, 서버상태=TanStack Query v5, UI상태=Zustand, 세션=Supabase 클라이언트, 로케일=`Intl` ko-KR. shadcn 80% 컴포넌트(Button·Dialog·Sheet·Popover·Command·Tabs·Toast·DropdownMenu·Separator·Avatar 등)는 그대로 쓰고, 이 문서는 **임상 특화 동작과 델타**만 규정한다.

**플랫폼 = 윈도우 우선.** 한국 중소병원 데스크(원무·진료실 PC, 종종 구형)는 거의 윈도우. 웹앱이라 크로스플랫폼은 무료지만 디테일을 윈도우 기준으로 명시한다(단축키 `Ctrl`, 폰트 폴백 맑은 고딕 우선, DPI 125–150%·1366×768 내성). 라우팅은 `kuntae802.mooo.com/patient_management_system` 서브패스 → Next `basePath` (Supabase Auth redirect · CORS · Flutter 웹뷰 base URL에 전파).

**설계를 끌어당기는 두 힘 (동시 만족 — 이 문서 전체의 합격선):**
1. **신규 직원(암묵지 0) 친화** — 모든 화면은 "이 사람이 선배 없이도 제대로 일할 수 있는가"로 판정한다. 그래서 **"다음에 할 일"(next-action) 어포던스가 1급 UX 패턴**이다.
2. **베테랑 속도 유지** — 강제(파이프라인·마스터 피커)가 베테랑을 느리게 하거나 free-text 우회를 유발하면 실패다(브리프 §8.1 반대지표). 가이드는 *방해 없이* 절제되어야 한다.

→ 시각 토큰·미학 자세·"운영 기억(operating memory)" 노스스타는 `DESIGN.md`(Brand & Style)를 본다. 다크 모드·모션은 **v1 보류**(라이트 전용, 절제된 마이크로 인터랙션) — 1차 구현 후 판독실/영상 수요 보고 재검토. 사실 아님, 결정 보류.

## Information Architecture

**전역 셸 (6역할 공통 상속, RBAC 노출 게이트):** `DESIGN.md`(Layout & Spacing)의 시각 셸 위에 동작을 얹는다.
- 좌측 **사이드바**(접이식, 펼침/아이콘) — 역할별 항목 그룹. `usePermissions` 훅이 권한 없는 항목은 **렌더하지 않는다**(숨김; 막힌 화면 X). 활성 항목 = 좌측 액센트 바 + teal.
- 상단 **탑바** — 병원명/브레드크럼, 전역 환자검색(`Ctrl K`), 날짜/시계(KST), 실시간 인디케이터, 알림 벨+배지, 아바타/역할 메뉴.
- 진료 허브 등 작성 폭이 중요한 화면은 사이드바를 **아이콘 접힘**으로 진입(hover 툴팁으로 라벨).

**역할별 사이트맵 (decision-log 확정):**

| 역할 | route group | 홈(기본) | 화면 |
|---|---|---|---|
| 원무/접수 | `reception` | 대기 현황 | 접수 · 예약 관리 · 환자 등록 · 환자 검색 · 수납 · 문서 출력 |
| 의사(진료의+판독의 겸임) | `doctor` | 진료 대기 | 진료 허브 · 판독 · 환자 검색 |
| 간호사 | `nurse` | 처치 워크리스트 | 활력징후 입력 · 간호기록 |
| 방사선사 | `radiology` | 촬영 워크리스트 | 영상 업로드 · 장비 관리 |
| 관리자/원장(실구매자) | `admin` | 대시보드 | 마스터(진료과·진료실·약품·진단코드·수가) · RBAC(역할·권한) · 근무 스케줄 · 감사 로그 |
| 환자(앱) | `(patient)` | 예약 또는 내 기록 | 로그인/가입 · 예약 · 내 진료내역/처방/검사결과 (RLS 본인만) |

**교차 요소 (cross-cutting):**
- **환자 상세/프로필 = 별도 풀페이지** (이력 타임라인). 비-의사 역할도 진입.
- **진료(Encounter) 허브** = 시스템의 master canvas. 한 내원이 SOAP·진단·처방·검사·처치오더·활력·수납을 묶는 스포크 허브. URL 예: `/encounter/{date}/{chart_no}`.
- **전역 환자 검색 = `Ctrl K`** 커맨드 팔레트(이름·차트번호·연락처). 외형과 직교한 인터랙션 레이어.
- **알림 = 탑바 벨 드롭다운 + "전체 보기"** 링크(별도 화면 아님).

**확정된 4개 IA 결정 (decision-log):**
1. **대기 현황판 = 원무·의사 공유** 화면. 간호·방사선은 **직역별 워크리스트**(RLS 필터: 진료과/진료실=큐, 직역=워크리스트와 일치).
2. **환자 상세 = 별도 풀페이지** (진료 화면 임베드 아님 — 비-의사 역할 진입 허용).
3. **알림 = 벨 드롭다운 + 전체 보기 링크** (전용 알림 페이지 없음).
4. **환자 앱 v1 범위 = 예약 + 본인 조회**로 좁게 (회원가입·본인인증은 시뮬레이션).

모달 스택은 1단계까지(서피스 위 Dialog/Sheet; 다이얼로그 위 다이얼로그 금지). → 구성 레퍼런스 = `mockups/`의 6개 파일: `key-waiting-board.html`(대기판) · `key-encounter-hub.html`(진료허브) · `key-patient-app.html`(환자앱) · `key-billing.html`(수납) · `key-appointment-calendar.html`(예약캘린더) · `key-rbac-matrix.html`(RBAC매트릭스). 각 패턴별 인라인 링크는 Component Patterns 행에 둔다. **충돌 시 이 spine이 우선.**

## Voice and Tone

마이크로카피. 브랜드 보이스·미학 자세는 `DESIGN.md`. 영문 식별자/enum은 그대로 두고, 한글은 그 위의 copy layer다(용어집 1:1).

**전역 규칙:**
- **100% 한국어 UI + 표준 출력 문서.** i18n 프레임워크 없음(단일 로케일 + `Intl`).
- **한글 용어집(`docs/glossary.md`) = 단일 진실.** UI 라벨은 영문 snake_case 식별자/enum 표시명 위의 한국어 표현이며 용어집과 1:1로 일치한다(예: `encounter=내원/진료`, `order=오더`, `fee_item=수가항목`). 임의 한국어 식별자 도입 금지.
- **오류:** 기계용 `code` = 영문(머신 리더블), 사용자용 `message` = 한국어 토스트. 오류 메시지·로그·토스트에 **내부정보/PII(raw 주민번호) 절대 노출 금지**.
- **금액 = KRW 정수**("원" 접미, 소수 없음, `tabular-nums`). 시각·날짜 = KST 표시.
- **"다음에 할 일" 안내 톤** — 명령형·행동 우선("다음 호출: …", "진료 완료 → 수납"). 신규 직원에게 *무엇을 하라*를 말한다.

**직원 화면 (5역할, 데스크톱):**
- 간결·임상·가이드형. "워크플로우 가이드, **not 입력 폼**." 라벨은 절차를 규정한다.
- 시간 표기 = **24시간**(예: `14:30`).
- 안전·신뢰 톤(놀이/소프트웨어 카피 제거). 위험·경고는 차분하지만 단호하게.

**환자 앱 (모바일):**
- 평이한 쉬운 말(따뜻·안심). **임상 용어 풀어쓰기**(예: 진단명 옆 "(혈압이 높은 상태)", "하루 1번, 아침 식사 후 한 알 드세요").
- 시간 표기 = **12시간**(예: `오후 2:30`).
- 프라이버시 선행 카피("이수진 님 본인의 정보만 안전하게 표시됩니다. 다른 사람은 볼 수 없어요" = RLS 신뢰 노트).

| Do | Don't |
|---|---|
| "다음 호출: 12번 이민준 · 12분 대기" | "다음 작업을 선택하세요" (모호) |
| 직원 "진료 완료 → 수납" / 환자 "예약 확정하기" | 같은 동작을 화면마다 다른 말로 |
| 환자 "고혈압 (혈압이 높은 상태)" | 환자에게 "본태성 고혈압 I10" 그대로 |
| 오류 `message`=한국어, `code`=영문 | 한국어 식별자/코드 즉흥 도입 |
| 24시간(직원) / 12시간(환자) 분리 | 환자에게 14:30, 직원에게 오후 2:30 |

## Component Patterns

동작 규정. 시각은 `DESIGN.md`(Components)의 `{components.*}` 토큰을 이름으로 참조한다. shadcn 기본 컴포넌트는 그대로.

컴포넌트 명칭은 `DESIGN.md`(Components)의 토큰명과 1:1로 일치시킨다(괄호 = 한글 표시명). 각 행 끝에 그 패턴을 보여주는 목업 파일을 인라인으로 링크한다(충돌 시 spine 우선 — IA 참조).

| 컴포넌트 | 동작 규칙 | 시각 토큰 |
|---|---|---|
| **waiting-list-row (대기 현황판)** | 기본 = **상태별 그룹 섹션**(섹션 헤더 = 점+컬러 상태명+카운트 pill), **활성도 순**(진행중→접수→예약→완료→취소). 완료/취소는 **접힘+muted**(시야에서 내림, 클릭 확장). 정렬 토글(활성도/호출/대기시간) = 기본 활성도. 각 행 우측 = **다음-액션 ghost 버튼**(상태에 맞는 1~2개: 접수=호출·진료시작 / 예약=접수 / 진행중=완료 처리). 짧은 단순 목록은 플랫 정렬로 충분. 상단 **"다음 호출" 히어로**가 외울 필요 없는 다음 환자를 제시. → `mockups/key-waiting-board.html`(상태 그룹·다음-액션·KPI). | `{components.waiting-list-row}` · `{components.status-badge}` · `{components.button-ghost}` |
| **status-badge (status system A3)** | 진료상태 5종을 **색 + 도형(글리프) + (취소만)취소선**으로 중복 인코딩. 작은 점 + **라벨 텍스트에 상태색**(배경 0). 같은 5색 체계를 결제상태·예약상태·급여구분에 **일관 재사용**(한 번 배운 색=의미가 전 화면 통용). **상태머신: 허용된 다음 전이만** 액션으로 노출(역행·건너뛰기 UI에 없음). | `{components.status-badge}` (mapping/glyph) |

## State Patterns

shadcn 위 동작 델타. (HTTP 매핑은 architecture 확정: 잘못된 전이=409 / 권한=403 / 검증=422.)

| 상태 | 처리 |
|---|---|
| **로딩** | `isPending` → **스켈레톤**(예상 레이아웃에 맞춘 행). **스피너 금지.** |
| **빈 상태** | 의미 있는 한국어 문장 + 단일 다음-액션 버튼(예: 대기판 "오늘 접수된 환자가 없습니다" + "＋ 환자 접수하기"). |
| **실시간 끊김(stale)** | 배너("연결 지연 · 실시간 갱신 멈춤", 마지막 갱신 시각) + "표시된 데이터가 최신이 아닐 수 있습니다" 주의 + **"다시 연결" 액션**. **신선도 임계(≤5초) 초과 시 호출·수행·완료·정산 등 중요 동작 버튼을 비활성(가드, "권장" 아님) + 강제 새로고침** — 임상 안전 패턴(스테일 실시간) 참조. 자동 덮어쓰기 X. |
| **검증 오류 (422)** | **필드 인라인** 메시지 + **해당 필드로 포커스 이동** + `aria-invalid`/`aria-describedby` 연결(AT 낭독). 예: 진단 주상병 미지정 → "주상병을 1개 지정해야 합니다". 클라 Zod(즉시 UX) → 서버 Pydantic(권위) → DB 제약(최종선) 3단. 주민번호 검증: 형식+생년월일+성별/세기자리(외국 5–8 허용)=HARD(차단), 체크섬=SOFT(경고만 — 비차단 polite 라이브 리전, 2020 개편). |
| **권한 거부 (403)** | 사이드바 항목은 **비노출**. 화면 내 액션은 **`aria-disabled`(포커스 가능) + 잠금(⊘) + 사유**(접근가능명/`aria-describedby`로 AT에도 전달, hover 툴팁 단독 금지 — "이 작업 권한이 없습니다 …"). 숨기지 않고 보여 **학습 유도**(신규 직원 원칙, AT 포함). |
| **잘못된 전이 (409)** | **토스트(assertive 라이브 리전) + "새로고침" 액션**(예: "이미 완료된 진료입니다 · 다른 단말에서 상태 변경됨"). 로컬 미저장 변경 보존. 동시 수행/정산 경합 = 낙관적 잠금/버전 → 409(임상 안전 패턴 참조). |
| **파괴적/민감 동작** | 환자 삭제(소프트)·수가 조정·민감 권한 토글·고위험 비가역 쓰기(진료 완료·처방 발행·수납 확정)는 **확인 다이얼로그 필수**(신원/대상 재진술 + 환자 삭제·수가 조정은 사유 캡처) + **감사 스냅샷(전/후)**. 비민감 동작은 즉시 실행 유지. |

**실시간:** `postgres_changes`(RLS 필터: 진료과/진료실=큐, 직역=워크리스트; **select-list는 암호화/민감 컬럼 제외** — PII 패턴 참조) → TanStack Query 캐시 invalidate(`useQueueRealtime`·`useWorklistRealtime`), 신선도 **≤5초**. 쓰기/명령=FastAPI(Bearer), 단순 조회=Supabase 직접(RLS), 복잡 집계·문서=FastAPI. **mutation 중 버튼 disable**(이중 제출 방지 = 처치 중복방지 FR-093의 UX 1차선). **optimistic 업데이트는 안전한 곳에만** — 상태 전이·정산·처치 수행처럼 충돌 위험이 있는 곳은 **비낙관적**(서버 제출 시 재검증) + **실시간 수신 시 해당 행 즉시 잠금**(스테일 행에서 두 번째 수행을 막음).

## Interaction Primitives

**키보드 우선 — 단, 이것은 외형과 직교한 인터랙션 레이어다.** (decision-log: 키보드 속도 = EXPERIENCE 영역이며 화면 구성·미학과 별개로 설계.) 레거시 한국 EMR의 회색 난잡·과밀 팝업은 버리되, 그들의 **키보드 속도는 보존**한다(브리프 §8.1 반대지표 — 베테랑을 느리게 하지 않기).

- **전역 커맨드 팔레트 = `Ctrl K`** — 환자 검색(이름·차트번호·연락처)을 핵심으로, 빠른 이동·동작. shadcn `Command` 기반. 결과는 `aria-live`로 업데이트 알림.
- **단축키 표기 = `Ctrl`(NOT ⌘)** — 주 타깃 윈도우. 인쇄=`Ctrl P` 등. 힌트는 `kbd` 칩으로 노출.
- **"다음에 할 일"(next-action) 어포던스 = 1급 패턴** — 각 임상 단계 화면이 현재 내원의 **허용된 다음 작업만** 명시(대기판 행 버튼, 진료 허브 액션 바·flow stepper, 수납 "결제 완료 → 문서 출력"). 신규 직원이 절차를 외우지 않아도 시스템이 안내.
- **포커스 순서/탭 이동 = 읽기 순서와 일치.** `Esc`는 항상 최상위 모달/팝오버를 닫는다. 422 시 첫 오류 필드로 포커스 이동.
- **실시간 구독** = 사용자 동작 없이 큐/워크리스트가 ≤5초 내 갱신(위 State Patterns).

## Accessibility Floor

동작 규정. 대비 수치 등 시각값은 `DESIGN.md`(Colors·Elevation & Depth)에 있다.

- ⭐ **색만 / 음영만으로 의미 전달 금지.** 색약 사용자 + 저가·미보정 임상 모니터를 **동시에** 대응하기 위해, 모든 중요 신호는 **색 + 도형(글리프) + 굵기**로 중복 인코딩한다. 상태=점+글리프+컬러 라벨(취소는 취소선), 슬롯=채움+테두리+패턴, RBAC 허용=채움+✓, 안전 경고=채움+테두리+굵은 라벨. 미세 box-shadow·옅은 틴트 단독에 의미를 싣지 않는다(현장 모니터에서 사라짐).
- **시맨틱/ARIA (네이티브 우선)** — 진짜 `<button>`/`<input>`/`<a href>`로 구현(`<span>`/`<div>` 인터랙션 금지). RBAC 매트릭스 = `<table>` + `<th scope>`(행·열 헤더 연결, 셀 접근가능명 = "역할 — 권한 — 허용/차단"). 캘린더 = `role="grid"`(슬롯 = gridcell). 대기판/워크리스트 = 리스트 시맨틱. **아이콘 단독 버튼은 전부 접근가능명**(`aria-label`; `title`·CSS 툴팁에 의존 금지) — 벨·사이드바 토글·접힘 내비·날짜 화살표·삭제(×) 포함.
- **라이브 리전 / 알림 (안전 임계)** — 알레르기/안전 경고 = `role="alert"` / `aria-live="assertive"`(화면 최우선 안전 신호는 반드시 낭독). 409 충돌 토스트 = assertive 라이브 리전. autosave 인디케이터 = `aria-live="polite"`. 422 검증 = 오류 메시지를 필드에 `aria-describedby`로 연결 + `aria-invalid`, 필수 필드는 `aria-required`(색 외에도 "(필수)"로 표시). 실시간 행 갱신 등 의미 있는 상태 변화는 polite 리전으로 알림.
- **주민번호 reveal 접근성** — "표시" 컨트롤은 접근가능명을 가진다(예: "주민등록번호 표시 — 조회 시 감사 로그 기록됨"). reveal 시 변경(복호화된 값 출현)이 라이브 리전으로 낭독되고, **감사 기록된다는 사실이 AT에 전달**된다(접근가능명 안에 감사 경고 포함). 연락처 등 모든 PII reveal에 동일 적용.
- **키보드 완전 조작** — 모든 동작이 키보드로 가능. 커맨드 팔레트·다이얼로그·테이블 행 액션 포함. **논리적 탭 순서**(읽기 순서 일치), 트랩 없음, 전 기능 조작 가능. **복합 위젯 = roving-tabindex(또는 화살표) 키보드 모델**: SOAP ledger(S→O→A→P + 파트 액션), slot-grid(시간×의사 2D 화살표 + Enter 예약), RBAC 매트릭스(110셀 2D 화살표, Tab 순회 금지). 모달/시트/peek = 포커스 트랩 + 초기 포커스 + 닫을 때 포커스 복원(상속이 아니라 요구사항으로 명시).
- **포커스 가시성** — `:focus-visible` 포커스 링(`{colors.ring}`) 항상 보이게(인접 보더 대비 충족 — DESIGN.md 대비 정책). SOAP 입력 중 좌측 teal 액센트는 추가 affordance.
- **403 비활성 = 사유 낭독 가능** — 비활성 액션은 `disabled`(포커스 불가) 대신 `aria-disabled`로 포커스 가능하게 두고, 사유를 접근가능명/`aria-describedby`로 제공(hover 툴팁 단독 금지) — "학습 유도"가 AT 사용자도 포함하도록.
- **모션** — `prefers-reduced-motion` 존중(시머·슬라이드·글로우 제거, 즉시 상태 전환). v1 확정(Open Items의 모션 미세조정과 별개로 이 플로어는 지금 약속).
- **환자앱 터치 타깃 ≥ 44px** — 슬롯·버튼·탭·날짜 칩 모두. 넉넉한 탭 간격. 데스크톱 인터랙티브 타깃은 ≥24×24px 유효 히트 영역(패딩 포함).
- **환자앱 줌·동적 타입** — 브라우저 줌 200%에서 콘텐츠가 reflow(고정 px로 줌 차단 금지, 상대 단위), **최소 본문 폰트 플로어** 보장, OS 동적 타입 존중(58세+ 사용자가 시스템 글자 크기를 키워도 깨지지 않게).
- **대비** — 본문/라벨/상태 라벨이 배경 대비 **DESIGN.md의 WCAG AA 대비 정책(대비 타깃)** 을 충족(본문 4.5:1 · 큰 텍스트/UI 3:1; 수치·검증 조합은 DESIGN.md `{colors}` 대비 정책). 의미를 전달하는 텍스트에 미달 토큰(`text-disabled`) 사용 금지. 환자앱은 데스크톱보다 큰 타입.

## Inspiration & Anti-patterns

- **영감 — Linear:** 밀도 규율, 1px 헤어라인 경계·낮은 elevation, 단일 액센트 절제, 상태=작은 컬러 점/라벨, **커맨드 팔레트** 속도감, 즉각·절제된 마이크로 인터랙션. 상태별 **그룹 리스트 패턴**(섹션 헤더+카운트, 완료 접기)도 Linear DNA.
- **안티패턴 — 레거시 한국 EMR:** 회색 난잡·과밀 팝업은 버린다. **단, 그들의 키보드 속도는 보존**(반대지표). 모던함은 의도된 선택(브리프: 비-EMR, 신규 직원 중심)이되 베테랑을 느리게 하면 안 됨.
- **거부 — 균일 카드화(v2):** 모든 섹션을 균일 elevation/카드로 박싱하는 안은 "과함"으로 철회됨. 영역 분리는 **중요도 위계 + 헤어라인 + 톤 레이어링**으로(예: SOAP는 테두리 없는 열린 캔버스로 대비 강조). 상세는 `DESIGN.md`(Elevation & Depth).
- **거부 — 음영 의존:** 위 Accessibility Floor 참조.

## 임상 안전 패턴 (Clinical Safety)

- **알레르기/안전 경고 = 누락 0 (can't-miss).** 환자 배너 상단에 상시 노출, **색 채움 + 테두리 + 굵은 라벨 + danger 채움 아이콘**(음영 비의존, `{components.allergy-alert}`). 알레르기·상호작용 위험(예: 와파린 병용)을 칩으로. 오더 발행 화면에서도 가시. AT엔 `role="alert"`로 낭독(접근성 플로어).
- **알레르기↔오더 교차검증 (체계적 패턴).** 오더 발행 시점에 시스템이 그 약품을 환자의 **기록된 알레르기 AND 활성 투약(상호작용)** 에 대조한다 → **하드 블록**, 또는 **사유 기록 오버라이드 + 감사**. 한 목업의 손으로 둔 경고 칩이 아니라 모든 처방·오더에 적용되는 1급 패턴(와파린/NSAID·아스피린 알레르기 사례를 포함하며, 현재 존재하는 건 동일성분 중복 FR-052뿐). `{components.order-panel}`에서 동작.
- **동일성분 중복 처방 경고**(FR-052) — 처방 추가 시 인라인 경고.
- **알레르기 경고 오버플로** — critical 알레르겐은 **truncation/"더보기" 뒤로 절대 숨기지 않는다.** critical 항목은 전부 노출 + 카운트(확장 가능), 어떤 안전 항목도 조용히 접히지 않는다. 중증도 순으로 먼저 렌더, 알레르겐을 collapse하는 것은 금지.
- **이미 수행된 오더 재수행 차단**(FR-093) — 수행 완료 오더는 액션 비활성·잠금(중복/누락 0; 목업에도 수행완료=잠금 상태 노출). mutation 중 버튼 disable이 1차선, 상태머신이 최종선.
- **오더 누락/지연 감지 (누락 0 디텍터).** 오더는 라이프사이클을 가지며(오더→수행→완료), **미수행/지연 오더는 워크리스트 + 진료 허브에 가시 인디케이터**(연령/임계치)로 surface된다 — 아무것도 조용히 사라지지 않게. 행별 상태 텍스트만으로는 디텍터가 아니다.
- **오더 → 수행 추적** — 지시자·수행자·시각이 항상 보인다("누가/언제/무엇을"). 워크리스트(간호·방사선)는 직역별로 자기 할 일만(RLS).
- **신원 확인 (고위험 비가역 쓰기).** **진료 완료 / 처방 발행 / 수납 확정 / 환자 삭제** 전에 환자 신원을 재확인한다(상시 환자 배너 + confirm 단계에 이름+차트번호 재진술). 신규 직원이 "잘못 열린 탭"에 작성·확정하는 오류를 직접 막는다.
- **다중 진료/스테일 탭 가드.** 세션당 활성 내원 컨텍스트는 **1개**. 스테일하거나 비-포그라운드 내원에 작용하려 하면 경고/차단(autosave도 거부), 포커스 복귀 시 reconcile. (409 전이 충돌과 별개로, 잘못된 환자의 열린 SOAP에 조용히 autosave되는 것을 막는다.)
- **동시 편집 경합.** 동시 수행/정산은 **낙관적 잠금/버전**으로 보호 — 충돌 시 409 + 명확한 복구. 수행·정산 액션은 비낙관적이며 서버 제출 시 재검증, 실시간 수신 시 행 잠금(FR-093과 연결).
- **스테일 실시간 = 쓰기 가드.** 실시간 채널이 신선도 임계(≤5초)를 넘어 스테일하면 호출·수행·완료·정산 등 중요 동작 버튼을 **비활성(경고/차단 + 강제 새로고침)** — "권장"이 아니라 가드. 자동 덮어쓰기 X.

## PII · 감사 패턴 (Privacy & Audit)

- **주민번호 기본 마스킹** + **권한 게이트 reveal** + **조회 자체가 감사 이벤트**. reveal 컨트롤에 "감사기록" 라벨 명시(누르면 기록됨을 사용자가 인지; AT엔 접근가능명으로 전달 — 접근성 플로어). raw 주민번호는 로그·토스트·오류 envelope에 절대 안 나간다. 복호화는 권한 백엔드 경로(service_role SECURITY DEFINER RPC)로만. **리스트/그리드/검색 결과도 기본 마스킹**(reveal은 상세에서만, 항상 감사; 리스트 행에는 per-row reveal 없음).
- **PII reveal 일관성** — 연락처·주소·보험·보호자 등 **모든 PII reveal은 주민번호와 동일하게** 권한 게이트 + 감사 + 접근가능한 "감사기록" 라벨을 가진다(필드별로 따로가 아니라 이 한 규칙으로 통일).
- **인쇄/PDF PII 정책 (신규).** 인쇄/PDF 문서(영수증·계산서·세부내역서)는 **주민번호 기본 마스킹**; full 주민번호는 권한 + 감사 + 사유가 있을 때만 렌더(문서에 full 주민번호를 찍는 것 자체가 권한 게이트된 **감사 reveal 이벤트**). **PDF/인쇄 파일명에 PII 금지**(chart_no/불투명 id 사용). 민감 문서 인쇄/내보내기 자체가 감사 이벤트. 법정 서식별 노출 필드는 양식 요건에 따라 명시.
- **PII 경계 확장** — raw 주민번호/PII는 **로그·토스트·오류 envelope에 더해 URL·딥링크 파라미터·실시간(postgres_changes) 페이로드·PDF/파일명·클라이언트 로그에도** 금지. 라우트엔 chart_no/불투명 id 사용(내원 URL은 주민번호를 담지 않는다 — `/encounter/{date}/{chart_no}`). 실시간 구독 select-list는 암호화/민감 컬럼을 제외(RLS는 행 스코프이지 컬럼 스코프가 아님). 환자 포털 조회는 항상 세션 uid로 스코프(클라이언트 제공 patient_id 미수용).
- **RLS 데이터 스코핑** — 환자=본인만, 직원=역할별. DB 강제 → 화면은 범위 밖 데이터를 **절대 받지 않는다**(성공지표: "환자 본인 외 데이터 접근 0"). 환자앱 신뢰 노트로 이를 사용자에게 전달.
- **감사 로그 = append-only** — 관리자 화면에서 조회·필터(행위자·기간·대상)만. 편집/삭제 불가(읽기전용 diff 뷰어). 변경 전/후 스냅샷. 감사 포착 범위 = 모든 PII reveal(주민번호+기타) · 모든 인쇄/PDF 내보내기 · 모든 상태 전이 · 모든 RBAC 변경 · 모든 파괴적 동작(관리자 본인의 reveal/조회도 예외 없이 기록).
- **RBAC 변경 = 즉시 적용 + 감사 기록**(비민감 토글). **민감 권한은 확인 단계 필수 + 감사**(저장 버튼 없는 즉시-적용은 비민감에만, 위 permission-cell).

## 상태머신 반영 (State Machine Mapping)

- **노출 규칙:** UI는 상태머신이 **허용한 다음 전이만** 액션으로 보여준다. 내원(예약→접수→진행중→완료, +취소/노쇼), 검사(오더→촬영완료→판독완료), 오더(오더→수행완료→취소), 예약(확정/취소/노쇼/완료), 결제(미수납/부분/완료). 역행·건너뛰기 버튼은 존재하지 않는다.
- **오류 매핑(architecture 확정):** 잘못된 전이 = **409**(토스트+새로고침) / 권한 = **403**(비활성+사유) / 검증 = **422**(필드 인라인+포커스).
- **RBAC 3계층:** UI 노출(이 문서) / FastAPI 명령 강제(쓰기 권위) / RLS 행 강제(데이터 권위). UI 게이트는 보안 경계가 아니라 *학습·속도* 레이어다.

### Flow A — 정해린(입사 2주차 원무), 화요일 오후 2:30

신규 직원이 선임 없이 정확히 "다음 환자"를 호출한다.

1. 정해린은 `reception` 홈인 **대기 현황판**에 있다. 화면은 상태별 그룹(진행중→접수→예약…), 완료/취소는 접혀 있다.
2. 상단 **"다음 호출" 히어로**가 말한다: "12번 이민준 · 정형외과 2진료실 · 12분 대기." 그녀는 순번을 외울 필요가 없다.
3. 접수 그룹 12번 행의 우측 **다음-액션 버튼 "호출"**을 누른다(또는 히어로의 "호출"). 버튼은 mutation 동안 비활성(이중 호출 방지).
4. 실시간으로 행 상태가 갱신되고, 다음 환자가 히어로로 올라온다.
5. **Climax:** 입사 2주차의 해린은 누구를 부를지, 두 번 부르거나 건너뛰지 않을지 **한 번도 헷갈리지 않았다.** 화면이 "다음에 할 일"을 가리켰고, 그녀는 정확히 그것을 했다.

실패: 실시간 끊김 → stale 배너 + "다시 연결". 호출 전 새로고침 권장. 409(다른 단말이 이미 호출) → 토스트+새로고침, 로컬 상태 보존.

### Flow B — 김도현 과장(정형외과 의사), 진료 허브

베테랑이 한 화면에서 끊김 없이, free-text 우회 없이 진료·오더·완료한다.

1. 진료 대기에서 환자(이수진)를 진료 시작 → 내원이 **진행중**으로 전이, **진료 허브**가 열린다.
2. 좌 컨텍스트 패널에 (간호가 잰) **활력징후**, **임상 프로필**, **과거 이력 타임라인**이 한 화면에 — 탭 이동·과거 차트 뒤지기 없음. 배너 상단엔 **페니실린·아스피린 알레르기, 와파린 병용 경고**가 can't-miss로.
3. 중앙 **SOAP ledger**에 O행을 클릭하니 좌측 teal 액센트가 켜지고("여기 타이핑 가능"), 그는 빠르게 기록한다. 진단은 **KCD 피커**로 I10·E11.9를 골라 주/부상병 토글(free-text 아님, 그래도 빠름).
4. 우 오더 패널에서 약품·검사를 **검색 피커**로 추가 — "베테랑만 알던 코드"를 몰라도 마스터가 채워준다. 수가는 "자동 산정"으로 프리뷰. 권한 밖(마약류) 추가는 비활성+사유 툴팁.
5. 하단 액션 바의 flow stepper(접수▸진료▸**완료**▸수납)에서 **"진료 완료 → 수납"**. autosave는 이미 기록을 지켰다.
6. **Climax:** 김 과장은 **한 화면에서 끊김 없이** 보고·기록·오더·완료를 마쳤다. 강제(피커·상태머신)는 그를 느리게 하지 않았고 free-text로 우회할 이유도 없었다.

실패: 주상병 미지정으로 완료 시 **422** → 진단 필드로 포커스 + "주상병을 1개 지정해야 합니다". 동일성분 중복 처방 시 인라인 경고.

### Flow C — 이수진(32세 환자), 앱으로 예약 → 내원 → 결과 확인

본인 데이터만(RLS), 쉬운 말로.

1. **예약 탭:** 진료과(정형외과) → 의사(김도현 과장) → 날짜 칩 레일(휴진일 비활성) → 시간 슬롯 그리드에서 **오후 2:30**(마감 슬롯은 비활성). 선택 요약 확인 후 "예약 확정하기"(≥44px CTA). (SMS 리마인더 3일·1일 전 = 시뮬/로그.)
2. **내원일:** 원무가 접수, 진료(Flow B), 수행·수납을 거친다.
3. **내 기록 탭:** 진료내역 카드를 펼치니 진단 "고혈압 (혈압이 높은 상태)", 처방("하루 1번, 아침 식사 후 한 알"), 검사 결과 요약(혈압 142/90 "주의", 콜레스테롤 "정상").
4. 상단 **신뢰 노트**: "이수진 님 본인의 정보만 안전하게 표시됩니다. 다른 사람은 볼 수 없어요."
5. **Climax:** 이수진은 전화 없이 예약했고, 진료 후 자신의 결과를 **쉬운 말로, 본인 것만** 확인했다 — 데이터 프라이버시가 그녀의 주의력이 아니라 시스템(RLS)으로 보장된다.

### Golden Path — UJ-1 (6역할 가로지르기)

예약(앱·이수진) → 접수(원무·정해린) → 진료 대기→진료 허브(의사·김도현) → 수행(간호 한지우 활력·처치 / 방사선 오민재 촬영·판독) → 수납·문서 출력(원무·정해린) → 결과 확인(앱·이수진). 관리자(최원장)는 권한 토글·대시보드·감사로그로 운영을 본다. 각 역할은 자기 화면에서 **외부 안내 없이** 자기 단계를 완결하고, 다음-액션이 인계를 끊김 없이 잇는다. 이 흐름이 **"끊김 없이" 완주되는가**가 제품의 합격선이다.
