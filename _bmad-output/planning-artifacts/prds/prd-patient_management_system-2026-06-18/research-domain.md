# 한국 중소병원 외래 운영 도메인 리서치 Digest

> 출처: PRD 작성용 백그라운드 도메인 리서치(2026-06-18). FR 현실감 보강 근거.

## 1. 외래 환자 동선/워크플로우
- 표준 흐름: ① 접수/수속 → ② 선수납(또는 진료 후 수납) → ③ 진료과 대기·순번 호출 → ④ 의사 진료 → ⑤ 진료 후 안내(검사예약·약처방·다음 외래) → ⑥ 수납(창구/무인기) → ⑦ 투약/주사/검사 수행 → ⑧ 원외처방약 수령. (한양대병원 진료절차)
- 직역 분담: 원무(접수·수납·청구·제증명·미수금), 의사(진료·진단·처방·오더), 간호사(활력징후·처치/주사·간호기록), 방사선사(영상 촬영), 약사(원내 조제).
- **선수납 vs 후수납** 분기가 운영 정책상 실제 존재(대기시간 개선 사례).
- **OCS(처방전달시스템)** 가 흐름의 중추: 의사 처방을 검사실·약국·간호·방사선(RIS)으로 전달.

## 2. 표준 진료비 문서
- 근거: 「진료비 세부산정내역 서식 등에 관한 기준」 고시, **2018.3.2. 시행**.
- 계산서·영수증 대분류: 진찰료 / 입원료 / 식대 / 투약 및 조제료 / 주사료 / 마취료 / 처치 및 수술료 / 검사료 / 영상진단 및 방사선치료료 / 치료재료 / 정신요법료 등.
- 금액 구분: 급여(본인일부부담/전액본인부담, 각 항목 **본인부담 vs 공단부담** 분리) / 비급여. 합계행: 본인부담총액 / 이미 납부한 금액 / 납부할 금액.
- 세부산정내역서 라인 필드: {항목분류, 일자, **코드**, 명칭, 단가, **횟수**, **일수**, 금액, 본인부담, 공단부담}. 최초 1회 무료 발급 의무.

## 3. 코드 체계 (마스터 데이터)
- **KCD(진단)**: 영문1+숫자, 다단계(대22/중267/소2,093/세12,603…), 3~5년 주기 개정(8차 2021→9차 2026). 심평원 상병마스터.
- **EDI 수가코드(행위)**: 심평원 건강보험 행위급여목록. {수가코드, 명칭, 분류번호, 급여여부, 단가}.
- **약품 코드(이원)**: 주성분코드 9자리(성분·함량·경로·제형 → 대체조제 가능) + 의약품 표준코드(KD) 13자리(약가마스터, ATC 매핑 별도).

## 4. SOAP 노트 (외래)
- S(주관적: 호소·병력) / O(객관적: 신체검사·검사수치·영상) / A(평가: 종합 → 진단명/잠정진단) / P(계획: 처방·검사·추후방문).
- **A↔P 연계(핵심)**: Assessment의 진단(KCD)이 Plan의 처방·오더 근거 → EMR/OCS에서 진단→오더 전산 연계, 기록 즉시 작성 원칙.

## 5. 역할별 화면 (국내 EMR)
- 시장: 의사랑(유비케어, 1위)·비트U차트(비트컴퓨터)·전능아이티 3강. 공통 모듈: 접수·진료·검사·삭감점검·수납·청구·통계 통합.
- 원무: 접수/수납/청구/제증명. 의사: 차트(SOAP)·진단·처방오더 + PACS 즉시 조회. 간호사: 활력·처치·주사·간호기록 + 오더 수신. 방사선사: RIS 워크리스트 + PACS.
- **간호 EMR에 기록만 정확히 하면 행위료/치료재료 수가가 자동발생**(별도 전표 불필요).

## 6. 예약/접수 운영
- 노쇼율 **11~25%**. 영향요인: 방문횟수·이전 노쇼 이력·낮은 연령·미혼·겨울·2주+ 대기.
- 관리: 예약금 환불불가(61.7%)·취소수수료·예약제한. **리마인더 SMS(62%), 최적 3일 전 1회 + 1일 전 1회**. 노쇼 2회+ 시 예약 제한 운영 병원 존재.
- 오버부킹/더블부킹은 노쇼 보전 vs 대기 증가 trade-off. 휴진·응급·진료지연이 정시 진료 실패 주원인.
- 의사 근무표/휴진 예외 처리 구체 관행은 공개자료상 **불확실**(휴진 시 예약 재배정/안내가 실무 이슈로만 언급).

## 7. 온보딩/세대교체
- 원무 업무 복잡(청구·미수·대불·산재·자보 등)으로 신규 직원 1개월+ 적응난 통념.
- 병원행정 직무는 NCS로 정의되나 실제 인계는 S-OJT 매뉴얼 의존 → 표준 매뉴얼 부재 시 암묵지 전수 공백.
- 정량적 이직률/세대교체 데이터는 **불확실**.

## PRD 반영 포인트
1. 워크플로우 상태기계: 예약→접수→대기/호출→진료→오더→처치/검사→수납을 명시적 status로. **선수납/후수납 분기**를 정책 플래그로.
2. 진료비 영수증 출력: 대분류 항목 × (급여 본인/공단, 비급여) + 합계 3행(본인부담총액·기납부·납부할금액). 표준서식 준수.
3. 세부산정내역서 출력: 라인 = {항목분류, 일자, 코드, 명칭, 단가, 횟수, 일수, 금액, 본인/공단부담}.
4. 마스터 3종: KCD 진단 / EDI 수가 / 약품(주성분9+KD13). 모두 **버전·유효기간** 관리.
5. SOAP A(진단 KCD) → P(오더) FK 연계, 수가 자동발생 근거.
6. 직역별 RBAC 화면 + 간호기록→수가 자동발생 규칙.
7. 예약: 노쇼 이력 카운트, N회 시 제한, SMS 리마인더(3일/1일 전), 오버부킹 정책, 휴진 재배정.
8. 온보딩: 업무 매뉴얼/체크리스트 + 작업 이력 로그로 암묵지→명시지.
