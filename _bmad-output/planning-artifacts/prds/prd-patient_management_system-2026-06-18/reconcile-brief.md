# 브리프 → PRD Reconciliation: 커버리지 갭

> 임무: 브리프에 있으나 PRD가 **누락·약화·왜곡**한 것 (특히 정성적 요소). PRD가 잘 담은 것은 나열하지 않음.

## A. 누락 (Missing) — 브리프에 있으나 PRD에 흔적이 없음

### A1. "정직한 차별점" 섹션 전체가 PRD에 없다
- **브리프 위치:** "무엇이 다른가 — 정직한 차별점과 한계" (62~80행). 차별점 3개: ①문제를 다르게 정의함(전산화가 아니라 암묵지→명시지) ②단순화 대신 정석 ③입력 폼이 아니라 워크플로우 가이드. 그리고 관통 정체성: **"모호한 요청을 현실적 시스템으로 구체화하는 능력의 전시"**.
- **PRD 상태:** 차별점 3개 중 **어느 것도 명시적 항목으로 존재하지 않음**. ③(워크플로우 가이드)만 NFR-050에 기능 요건으로 희석되어 살아남음. ①②와 "구체화 능력 전시"라는 프로젝트 정체성 자체가 통째로 증발.
- **PRD 어디에 들어가야 하나:** §1 개요에 "차별점/포지셔닝" 단락 신설, 또는 §6.1 ②과제 성공 신호 위에 "포지셔닝" 소절 추가. 특히 "기능이 아니라 문제를 보는 각도가 차별"이라는 프레이밍.

### A2. "솔직한 한계" 3개 중 2개가 NFR로만 부분 흡수, 나머지는 누락
- **브리프 위치:** 74~80행. 한계 3개: ①소프트웨어는 만병통치 아님(워크플로우 강제 = 현장 유연성 감소, 교육·문서화 여전히 필요) ②학습용 모사(인증·규제·EDI 미충족) ③폭·깊이 트레이드오프(단일 개발자·일정 → 기능 깊이 제한). 그리고 메타 문장: **"한계를 숨기지 않는 것 자체가 프리랜서 신뢰를 만드는 방식"**.
- **PRD 상태:** ②만 NFR-022·§6.2에 살아남음. ①(유연성 감소·교육 병행 필요)은 §6.1 카운터 지표에 *증상으로만* 부분 반영(우회·자유 입력 요구)되고 한계로서의 솔직한 인정은 없음. ③(깊이 제한)은 완전 누락. "한계 공개=신뢰" 메타 프레이밍도 누락.
- **PRD 어디에 들어가야 하나:** §6.2 범위 아래 "솔직한 한계 / Known Limitations" 소절 신설. ①의 "교육·문서화 병행 필요"는 NFR-050/051(신규 직원 친화) 근처에 단서로 명시.

### A3. "설계의 중심 인물 = 신규 직원" 서사가 원리에서 기능 문장으로 격하
- **브리프 위치:** 82~98행. "설계의 한가운데에 한 사람이 있다: 아직 암묵지를 체득하지 못한 신규 직원. 모든 화면은 '이 사람이 선배 없이도 제대로 일할 수 있는가'를 기준으로 만들어진다." + "베테랑은 시스템 없어도 일하지만, 신규 직원은 시스템이 안내해야 일한다."
- **PRD 상태:** NFR-050/051에 "신규 직원이 절차를 암기하지 않아도"로 *결과*만 남음. 그러나 이것이 **단일 설계 기준점(north-star persona)**이라는 위계가 사라짐 — PRD에서는 여러 사용성 요건 중 하나로 평평해짐. 6역할 사용자 설명(브리프 86~96행: 각 역할의 *필요/성공* 쌍)도 PRD에 없음.
- **PRD 어디에 들어가야 하나:** §1 개요 또는 §5 사용성 머리말에 "설계 중심 인물" 명문화. 6역할의 필요/성공 쌍은 §3 각 기능 그룹 머리말 또는 별도 "사용자·역할" 섹션으로.

### A4. 브리프의 "왜 지금(Why now)" = 세대교체 가속 서사가 PRD에서 정적 배경으로 약화
- **브리프 위치:** 26행 "조직의 운영체계가 사람과 함께 걸어 나가고 있다" / 36행 "세대교체가 가속될수록 커지는 비용" / 34행 "버스 팩터 1".
- **PRD 상태:** §1 배경이 "세대교체 국면에 들어섰다"로 *상태*만 서술. **시간적 긴급성("왜 지금")·가속 비용·버스 팩터 1**의 동기 서사가 빠짐(버스 팩터는 §6 지표 표에만 단어로 잔존, 문제 동기로서는 없음).
- **PRD 어디에 들어가야 하나:** §1 개요에 "왜 지금(Why now)" 한 문단.

## B. 약화 / 왜곡 (Weakened / Distorted)

### B1. 7기둥 ↔ 고통 1:1 매핑이 깨짐 — 가장 큰 구조적 손실
- **브리프 위치:** 44~60행. 7기둥 각각이 "→ *(겨냥 고통)*"으로 명시적 1:1 매핑. 이것이 브리프의 핵심 논증 구조(평가 포인트 ②"구조 변환"의 증거물).
- **PRD 상태:** 7기둥이 §3 FR 그룹으로 *분해·재배열*되며 **"이 FR이 어떤 고통을 푸는가"의 추적성이 사라짐**. FR은 동작만 서술하고 동기(왜)를 잃음. §6.1 ②에서 "고통↔FR/스키마 매핑"을 *주장*은 하지만, PRD 본문 어디에도 그 매핑이 *실제로 그려져 있지 않음*. 즉 PRD가 스스로 증거라 부른 것이 본문에 부재.
- **왜곡 포인트:** 기둥6(멀티플랫폼+환자포털)은 "예약 혼선·원무 과부하 해소"가 동기인데, PRD에서는 NFR-010/011 플랫폼 요건으로 흩어져 동기가 증발. 기둥1(파이프라인)의 "다음 할 일 안내"는 §2+NFR-050로 쪼개짐.
- **PRD 어디에 들어가야 하나:** §3 머리말 또는 §6.1 ② 아래에 **고통 → 기둥 → FR/NFR 추적 매핑 표** 신설. 이게 평가 포인트 ②의 실물 증거가 됨.

### B2. "운영 기억(operating memory)" 비전 + 종결 메시지 소실
- **브리프 위치:** 142~148행 비전 섹션. "궁극의 목표는 병원의 '운영 기억(operating memory)'이 되는 것" / "세대교체가 더 이상 위기가 아니라 그저 또 한 번의 평범한 인수인계가 된다" / "만든 사람에게: 다음 실제 프로젝트에 들고 갈 진짜 산출물".
- **PRD 상태:** **비전 섹션 자체가 없음.** NFR-060(입원 확장 수용)에 기술적 확장성만 남고, "운영 기억"이라는 제품 북극성 메타포와 종결 서사가 통째 누락. 입원 확장(브리프 144행: EDI·약국 전송·운영 대시보드로 깊이 더하기)도 PRD에서는 NFR-060의 한 줄로 축소.
- **PRD 어디에 들어가야 하나:** PRD 말미 "비전 / 향후 방향" 섹션 신설, 또는 §1 개요 마무리에 운영 기억 메타포 1문장.

### B3. 솔루션 관통 명제 "암묵지 → 명시지"가 PRD에서 흐릿
- **브리프 위치:** 60행 "일곱 기둥은 모두 같은 일을 한다 — 사람 머릿속의 암묵지를 시스템 안의 명시지로 옮긴다."
- **PRD 상태:** §1 배경에 "암묵지" 단어는 있으나, **"명시지로 옮긴다"는 솔루션의 단일 관통 명제가 솔루션 설계 원리로 격상되지 못함**. §2 머리말 "사람의 기억이 아니라 시스템에 새기는 척추"가 가장 근접하나 파이프라인에만 국한. 7기둥 전체를 묶는 명제로서는 부재.
- **PRD 어디에 들어가야 하나:** §1 제품 단락 또는 §3 머리말에 관통 명제 1문장 명문화.

## C. 모순 / 불일치 (Contradictions)

### C1. 역할 수 표기 불일치: "5역할" vs "6역할"
- **브리프:** 직원을 "5역할"(원무·의사·간호사·방사선사·관리자)로 명시(86행 "직원, 데스크톱 웹, 5역할"), 환자 포함 시 "6역할". 일관됨.
- **PRD:** §1에서 "6개 역할 화면", "6역할(원무·의사·간호사·방사선사·관리자·환자)"로 표기. 환자를 6역할에 포함 — 브리프와 동일 개념이나, 브리프가 구분한 **"직원 5역할 / 전체 6역할"의 위계가 사라짐**. 큰 모순은 아니나 환자를 직원 역할과 동렬 취급하는 미묘한 표기 드리프트. (영향 낮음, 단 RBAC FR-212에서 직원/환자 분리 프로필을 강조하므로 용어 일관성 권장.)
- **조치:** §1·§3에서 "직원 5역할 + 환자 = 6역할" 식으로 위계 복원.

### C2. 영상 판독 주체 — 브리프 vs PRD 결정 불일치 (잠재)
- **브리프:** 91행 방사선사 "오더받은 영상검사를 촬영" + "오더→촬영→판독 흐름". 판독 주체를 명시 안 함.
- **PRD:** FR-102 "의사(판독의)가 판독" + §6.3에서 "영상 판독 = 진료의 겸임 허용"으로 **확정**. 이것은 브리프에 없던 *새 결정*(다운스트림 finalize 산물)으로, 모순이라기보다 PRD가 추가 확정한 항목. 단 브리프 91행 방사선사 성공기준 "판독 흐름이 끊기지 않는다"와 정합하는지 확인 필요 — 정합함. (모순 아님, 기록용.)

### C3. 테이블 수: 브리프 "26개" 확정 vs PRD "약 26개"
- **브리프:** 117·128행 "26테이블", "외래 도메인 26개 테이블 전체"로 **확정 수치**.
- **PRD:** §1·§6.2 "약 26개"로 *완화*. 브리프의 확정 톤(정석을 다 넣었다는 자신감의 근거)이 약해짐. 메모리(pms-db-schema-brainstorm: 26개 테이블)와도 26 확정이 맞음.
- **조치:** 스키마 산출물과 대조해 26 확정이면 "약" 제거.

## D. 종합 우선순위 (부모 에이전트용)

1. **[최우선] B1** — 고통↔기둥↔FR 매핑 표 부재. PRD가 자기 평가근거(②구조변환)라 부른 것이 본문에 없음. 추적성 복원 필수.
2. **[높음] A1** — 차별점/"구체화 능력 전시" 정체성 통째 누락. 이 프로젝트의 존재 이유(평가 포인트 ①).
3. **[높음] A2 + B2** — "솔직한 한계"와 "비전(운영 기억)" 정성 섹션 부재. 브리프의 정직함·서사 양 끝이 잘림.
4. **[중간] A3** — 신규 직원 north-star persona가 NFR 한 줄로 평탄화.
5. **[중간] A4 + B3** — Why-now 긴급성 / 암묵지→명시지 관통 명제 약화.
6. **[낮음] C1, C3** — 용어·수치 드리프트(5/6역할, 약 26개).
