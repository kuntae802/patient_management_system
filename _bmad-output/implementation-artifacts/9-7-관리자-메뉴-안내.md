# Story 9.7: 관리자(admin) 메뉴 안내

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 관리자,
I want 내 관리 화면(운영/대시보드·마스터·권한·근무 스케줄·직원 계정·감사 로그)의 사용법을 실제 화면·번호 하이라이트·단계 설명으로 보기를,
so that 권한·마스터·근무·감사를 선배 없이 운영한다.

## Acceptance Criteria

1. **관리자 메뉴 전수 수록.** 관리자 계정으로 `/help`에 들어가면 사이드바 "관리" 섹션에 보이는 **6개 메뉴 전부**가 안내로 제공된다: **운영/대시보드 `/admin/dashboard`·마스터 `/admin/masters`·권한 `/admin/permissions`·근무 스케줄 `/admin/schedule`·직원 계정 `/admin/users`·감사 로그 `/admin/audit-logs`**. "준비 중" 플레이스홀더로 남는 관리자 메뉴가 없어야 한다.

2. **(FR-252) 각 메뉴 = 실제 화면 스크린샷 + 번호 하이라이트 + 설명표 + 흐름.** 6개 메뉴 각각 캡처 PNG + 번호 하이라이트(①②③…) + `hotspots` 설명표 + (선택)`flow`. 캡처 스펙 번호(n) = help-content `hotspots[].num` **1:1**. 각 메뉴는 단일 화면(1장)으로 안내한다(라디오·복합 패널 없음 — 6역할 중 화면 수가 가장 많은 역할이므로 화면당 1장으로 단순화).

3. **운영 포인트 설명(AC And 절).** **권한 토글 즉시 반영**(권한 화면: 셀 클릭 = 즉시 DB 반영·자동 저장)·**대시보드 월간/일별**(운영/대시보드: 기준일·집계 기간 선택으로 일별 추세·기간 합계)·**감사 가독성**(감사 로그: 읽기 전용·append-only·필터·페이지네이션·환자/보호자명 마스킹)이 각 메뉴의 intro/hotspot/flow에 명시된다.

4. **(9.2 파이프라인 소비) 캡처 자동 생성·정적 임베드.** 6개 화면 이미지는 `tools/screenshots/specs.mjs`에 admin 스펙 6개를 추가하고 `node capture.mjs admin`으로 생성해 **`web/public/help/admin/<screen>.png`**에 배치한다(런타임 캡처 0·외부 편집 도구 0). 캡처는 더미 데이터 + admin 계정(`admin@pms.local`·23권한 전부) 환경 대상. 원무(9.3)·의사(9.1/9.4)·간호(9.5)·방사선사(9.6) 콘텐츠·이미지는 무변경.

5. **회귀 무파괴·범위 한정.** 변경 = `web/src/lib/help/help-content.ts`(admin 가이드 6개)·`tools/screenshots/specs.mjs`(스펙 6개)·`web/public/help/admin/*.png`(생성)·테스트(`help-content.test.ts` 커버리지 + 선택 `help-guide.test.tsx` 렌더). **앱 화면 코드·API·마이그·권한·생성 TS 무변경.** `npm run build`/lint/타입체크 통과. 기존 도움말 셸·원무/의사/간호/방사선사 콘텐츠·사이드바 회귀 0.

> **범위 경계 (반드시 지킬 것):** 이 스토리는 **관리자(admin) 역할 콘텐츠 완성**까지다.

### ⚠️ 비자명·실수 방지
1. **🔴 캡처 중 mutation 금지(이 스토리 최대 함정).** 관리 화면의 토글·드롭다운·셀 클릭은 **클라우드 DB를 즉시 변경**한다: 권한 매트릭스 셀 클릭=실 grant PUT, 마스터 활성/비활성 토글=실 update, 직원 재직상태 드롭다운=실 update, 스케줄 토글=실 update. **6 화면 goto는 전부 "URL 이동 + 대기"만**(방사선사·간호처럼 행 클릭으로 패널 여는 화면이 **없음** — 전부 자동 로드). annotate는 토글/셀/드롭다운에 **번호만 부여**(boundingBox 잡힘)하고 클릭하지 않는다. 잘못 클릭하면 데모 DB 권한/마스터가 오염되어 후속 캡처·E2E가 깨진다.
2. **클라 fetch 3화면 = 대기 충분히.** 대시보드(`/v1/dashboard/operations`)·직원 계정(`/v1/admin/users`)·감사 로그(`/v1/admin/audit-logs`)는 **마운트 후 클라가 fetch** → 스켈레톤이 먼저 뜬다. goto에서 표/카드가 채워질 때까지 대기(waitForTimeout 2500~3500 또는 데이터 텍스트 대기). 너무 짧으면 스켈레톤이 캡처됨. 마스터·권한·스케줄(근무표 탭)은 RSC 서버 주입이라 즉시(단 스케줄 의사 목록은 클라 fetch).
3. **탭 화면(마스터·스케줄)은 기본 탭만.** 마스터 기본 탭=진료과, 스케줄 기본 탭=근무표. 탭 전환 없이 기본 탭을 캡처(다른 탭은 흐름 설명으로). 탭 버튼 자체에 번호를 줘 "5종 마스터/2탭" 안내.
4. **PII = 관리 화면은 대부분 운영·직원 데이터(환자 PII 희박).** 대시보드=집계(개인 0)·마스터=조직/코드·권한=역할/권한·스케줄=의사 근무·직원=직원 정보. **유일 주의=감사 로그**: 환자/보호자 대상 행은 서버 1차 + 클라 방어심층으로 **이름 마스킹(Lock·"—")**, 주민번호 0. 캡처는 안전하나 육안으로 마스킹 확인. 감사 상세 다이얼로그(보기)는 **열지 말 것**(번호는 "보기" 버튼에만).
5. **번호 정합·viewport·약한 locator.** n↔num 1:1. 1440×900(2단/표 상단부). 긴 표(권한 매트릭스·감사 로그)는 **상단에 보이는 요소**로 번호 부여(스크롤 밖 요소 금지). 약한 locator는 캡처로 검증(러너 "번호 0개" 경고 fail-soft).
6. **이미지 경로 = admin 영역.** `/help/admin/<key>.png`. 러너 role=admin → 기본 출력 `web/public/help/admin/`. screen 키 = 라우트 마지막 세그먼트(dashboard/masters/permissions/schedule/users/audit-logs).
7. **테스트 게이트 차이.** admin 메뉴는 `requiredPermission` 게이트가 **있다**(원무/의사/간호/방사선사 무게이트와 다름). 커버리지 테스트는 "권한 보유 관리자에게 보이는 6메뉴 = 전부 가이드"를 검증(roles.includes("admin") 필터 = 권한과 무관히 6개 href 추출 → 전부 HELP_GUIDES에). `filterNav` 권한 게이트 자체 검증은 staff-nav.test.ts 몫(중복 금지).
