## Story

As a **관리자**,
I want **의사별 근무표(요일·시간대·진료실)와 휴진·예외(휴가·학회)를 등록·수정·비활성하기를**,
so that **예약 가능 슬롯(FR-012)·휴진 재배정(FR-016)의 근거가 DB에 선다** (Epic 6 예약·스케줄의 토대).

> **에픽 위치:** Epic 6 첫 스토리(DB 토대). 이 스토리가 **근무 스케줄 sub-도메인의 스키마를 깐다**.
> 후속 스토리가 소비: 6.2(동적 가용 슬롯 계산 = 근무−예외−기예약), 6.3(예약 캘린더·더블부킹),
> 6.4(원무 대리 예약), 6.5(환자 앱 예약), 6.8(휴진 재배정). **예약(appointments) 테이블·예약 생성은
> 이 스토리 범위 밖**(§Scope Boundaries 참조).

## Acceptance Criteria

### AC1 — 근무표 등록·관리 (FR-220)
**Given** `0030_doctor_schedules.sql`이 `doctor_schedules` 테이블을 만들었을 때
**When** 관리자가 의사별 근무표(의사·진료과·진료실(선택)·요일·시작/종료 시각)를 등록하면
**Then** 근무표 행이 저장되고(`master.manage` 게이트·감사 기록), 같은 의사·같은 요일에 시간이 **겹치는** 활성 근무표는 거부된다(409 `schedule_overlap`).

### AC2 — 휴진·예외 등록·관리 (FR-221 → FR-012/FR-016 근거)
**Given** `doctor_time_offs` 테이블이 있을 때
**When** 관리자가 휴진·예외(의사·시작/종료 시각·사유[휴가·학회])를 등록하면
**Then** 휴진 행이 저장되어 **가용 슬롯 계산(6.2/FR-012)·휴진 재배정(6.8/FR-016)의 근거**가 되고, 감사에 기록된다.

### AC3 — soft delete · 참조 무결성 (마스터 2.1/2.4 패턴 정합)
**Given** 등록된 근무표·휴진 행에 대해
**When** 비활성(soft delete) 또는 수정하면
**Then** 물리 삭제 없이 `is_active`만 토글되고(과거 예약 참조 보존), 존재하지 않는/비활성 의사·진료과·진료실에 **신규 배정**하면 422(`invalid_doctor`/`invalid_department`/`invalid_room`)로 거부된다.

### AC4 — 권한 게이트 · RLS · 감사 (보안 횡단)
**Given** 근무표·휴진 쓰기 엔드포인트에 대해
**When** `master.manage` 미보유 직원(의사·간호·원무)이 호출하면
**Then** 403(`forbidden`)이고, 모든 직원은 근무표·휴진을 **읽을 수 있으며**(전역 참조 — RLS authenticated SELECT), 모든 변경은 `audit_logs`에 actor와 함께 append-only로 남는다.

### AC5 — 관리자 웹 화면 (`/admin/schedule`)
**Given** 관리자가 좌측 내비 "근무 스케줄"을 클릭하면
**When** `/admin/schedule` 페이지가 열리면
**Then** 근무표·휴진 탭, 의사·진료과·진료실 피커가 있는 등록/수정 폼, 비활성 확인 다이얼로그, 겹침/검증 에러 인라인 표시가 동작한다(마스터 관리 화면 패턴 미러).

### 데이터 모델 (정확 스키마)

**`doctor_schedules`** (근무표 — 주간 반복 패턴):
```
id            uuid PK default gen_random_uuid()
doctor_id     uuid NOT NULL → users(id)         -- 담당의(role=doctor; 서비스 검증)
department_id uuid NOT NULL → departments(id)    -- 슬롯 그룹핑(FR-010 진료과 선택). 의사가 여러 과 커버 가능 → 블록별 명시
room_id       uuid → rooms(id)                   -- 진료실(선택; 동적 배정 여지)
weekday       smallint NOT NULL CHECK (weekday between 0 and 6)  -- ⚠️ PG extract(dow): 0=일..6=토 (6.2 슬롯계산 정합)
start_time    time NOT NULL
end_time      time NOT NULL
is_active     boolean NOT NULL default true       -- soft delete(과거 예약 참조 보존)
created_at / updated_at  timestamptz NOT NULL default now()
CHECK (start_time < end_time)
CONSTRAINT doctor_schedules_no_overlap EXCLUDE USING gist (...)  -- 아래 SQL
```

**겹침 방지 EXCLUDE — 정확 SQL**(Postgres엔 내장 `timerange` 없음 → date-anchored `tsrange` 관용구로 immutable 보장):
```sql
create extension if not exists btree_gist with schema extensions;  -- uuid·smallint 의 gist '=' opclass

-- ⚠️ btree_gist opclass(gist_uuid_ops 등)가 extensions 스키마라, EXCLUDE 생성 시 opclass 미해석으로
--    "operator class gist_uuid_ops does not exist for access method gist" 실패 가능. search_path 명시.
set local search_path = public, extensions, pg_catalog;

alter table public.doctor_schedules
  add constraint doctor_schedules_no_overlap
  exclude using gist (
    doctor_id with =,
    weekday   with =,
    tsrange(('2000-01-01'::date + start_time),
            ('2000-01-01'::date + end_time)) with &&
  ) where (is_active);
```
- `date + time → timestamp`(immutable), `tsrange(ts,ts)` 기본 `[)`(immutable) → GIST 인덱스 적격.
- ⚠️ **`set local search_path`** 누락 시 opclass 미해석 실패 — 마이그 트랜잭션 상단(또는 ALTER 직전)에 반드시. 0005_crypto/seed가 `extensions.crypt`로 **함수는 스키마 한정**했으나 EXCLUDE opclass는 한정 문법이 까다로워 search_path가 실용적. 마이그 테스트가 제약 존재를 검증해 회귀 가드.
- `where (is_active)` 부분 제약 = 비활성(soft-deleted) 행은 차단 안 함(masters soft-delete 철학 정합). **재활성(set_active false→true)이 활성 겹침 유발 시에도 EXCLUDE 발화** → `set_doctor_schedule_active`도 `ExclusionViolationError` catch 필요.
- 위반 = `asyncpg.ExclusionViolationError`(SQLSTATE 23P01) → 서비스에서 catch → `ConflictError("...겹치는 근무표가 있습니다.", code="schedule_overlap")`. **`_map_pg_sqlstate` 변경 불요**(masters `UniqueViolationError`→`code_taken` 동형 — 서비스 레이어 catch).

**`doctor_time_offs`** (휴진/예외 — 특정 기간):
```
id          uuid PK
doctor_id   uuid NOT NULL → users(id)
start_at    timestamptz NOT NULL    -- 종일 휴가 = 자정~자정, 부분(학회) = 특정 시각
end_at      timestamptz NOT NULL
reason      text                    -- 저민감 운영 사유(휴가·학회). ⚠️ 임상/PII 자유텍스트 금지(cancel_reason 정합)
is_active   boolean NOT NULL default true
created_at / updated_at  timestamptz
CHECK (start_at < end_at)
-- 겹침 제약 없음(중첩 휴진 = 무해한 합집합; 슬롯 계산이 union 차감)
```
