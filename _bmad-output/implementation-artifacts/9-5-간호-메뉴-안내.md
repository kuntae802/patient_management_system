# Story 9.5: 간호(nurse) 메뉴 안내

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 간호사,
I want 내 간호 업무 화면(처치 워크리스트·활력징후 입력·간호기록)의 사용법을 실제 화면·번호 하이라이트·단계 설명으로 보기를,
so that 활력 입력 → 처치 수행(재수행 차단) 흐름을 이해하고 처치 중복·누락 없이 수행한다.

## Acceptance Criteria

1. **간호 메뉴 전수 수록.** 간호 계정으로 `/help`에 들어가면 사이드바에 보이는 **3개 메뉴 전부**가 안내로 제공된다: **처치 워크리스트 `/nurse/worklist`·활력징후 입력 `/nurse/vitals`·간호기록 `/nurse/notes`**. "준비 중" 플레이스홀더로 남는 간호 메뉴가 없어야 한다.

2. **(FR-252) 각 메뉴 = 실제 화면 스크린샷 + 번호 하이라이트 + 설명표 + 흐름.** 3개 메뉴 각각 캡처 PNG + 핵심 동작요소 번호 하이라이트(①②③…) + `hotspots` 설명표 + `flow`. 캡처 스펙의 번호(n) = help-content `hotspots[].num` 1:1. 세 화면 모두 **좌측 환자 목록 → 행 선택 → 우측 작업 영역(수행/입력)** 구조이므로, 캡처는 환자를 선택한 상태(우측 패널/폼 표시)로 한다.

3. **핵심 흐름 설명(AC And 절).** **활력 입력 → 처치 수행 → 재수행 차단** 흐름이 관련 메뉴 flow/intro에 명시된다: 활력징후를 기록하고, 지시된 처치를 "수행" 처리하며, **이미 수행된 처치는 다시 수행할 수 없다**(수행 완료 잠금). 간호사는 진료 허브 권한이 없어 이 세 워크리스트가 간호 업무의 진입점임을 설명한다.

4. **(9.2 파이프라인 소비) 캡처 자동 생성·정적 임베드.** 3개 화면 이미지는 `tools/screenshots/specs.mjs`에 nurse 스펙 3개를 추가하고 `node capture.mjs nurse`로 생성해 **`web/public/help/nurse/<screen>.png`**에 배치한다(런타임 캡처 0·외부 편집 도구 0). 캡처는 더미 데이터 + nurse 권한이 적용된 환경 대상. 의사(9.1/9.4)·원무(9.3) 콘텐츠·이미지는 무변경.

5. **회귀 무파괴·범위 한정.** 변경 = `web/src/lib/help/help-content.ts`(nurse 가이드 3개)·`tools/screenshots/specs.mjs`(nurse 스펙 3개)·`web/public/help/nurse/*.png`(생성)·테스트(`help-content.test.ts` nurse 커버리지 + 선택 `help-guide.test.tsx` nurse 렌더). **앱 화면 코드·API·마이그·권한·생성 TS 무변경.** `npm run build`/lint/타입체크 통과. 기존 도움말 셸·원무(9.3)·의사(9.1/9.4)·사이드바 회귀 0.

### 컨텍스트 요약
9.5는 간호 역할 콘텐츠를 채운다(9.3 원무·9.4 의사 패턴 반복). 간호 3메뉴 전부 신규(현재 플레이스홀더). 세 화면 모두 **좌 목록 → 행 선택 → 우 작업영역** 구조라 캡처 goto가 환자 선택까지 구동해야 우측이 보인다. 핵심 흐름 = 활력 입력 → 처치 수행(재수행 차단).

### ⚠️ 비자명·실수 방지
1. **세 화면 모두 행 선택 후라야 우측 표시.** goto가 좌측 환자 행을 클릭해야 우측 패널/폼(번호 대상 다수)이 렌더. 미선택 시 "좌측에서 환자를 선택하면…" 안내만 → 번호 누락.
2. **클릭 대상 = 데이터 있는 환자.** worklist=윤서아(유일하게 미수행 처치 t03 보유)·vitals=오세훈(활력 보유)·notes=오세훈(간호기록 1건). 다른 환자 클릭 시 우측이 빈약. 행 locator는 환자명 filter + 좌측 목록 첫 항목 폴백(9.4 패턴).
3. **⚠️ 클라우드 nurse 권한 선행(9.4 교훈 확장).** vitals=`vital.record`·notes=`nursing.record`·worklist=`treatment.perform` 가드. 로컬 nurse는 5권한 전부 보유(seed.sql)이나 **클라우드 role_permissions 가 동기화 안 됐으면** 페이지가 STAFF_HOME 리다이렉트 → 캡처 실패. 캡처 전 nurse 로그인 프로브로 확인, 미보유 시 그랜트(RBAC UI 또는 role_permissions 적용). 데이터(t03·활력·기록)는 9.4 클라우드 재시드로 이미 존재.
4. **좌측 헤더 텍스트 충돌.** 활력·간호기록 둘 다 좌측 헤더 "오늘 활성 내원". 각 화면 단독 캡처라 무방하나, help-guide 페이지엔 두 섹션이 함께 렌더되므로 테스트에서 `getByText("오늘 활성 내원")`는 복수 매칭 주의(getAllByText 또는 더 구체적 텍스트 사용).
5. **재수행 차단·"처치" 태그는 demo 기본 상태에 없음.** t03은 ordered만(수행 전). "수행 완료" 잠금 카드·간호기록 "처치" 태그는 t03 수행 후에야 보임 → **번호로 가리키지 말고 flow 텍스트로 설명**(9.4 "업로드 영상 없음" 선례). 캡처는 수행 전(수행 버튼 활성) 상태로.
6. **번호 정합·viewport·약한 locator.** n↔num 1:1. 1440×900(2컬럼·짧음=폴드 위). 약한 locator는 캡처로 검증(러너 "번호 0개" 경고 fail-soft).

- 데이터: t03(e08 윤서아·네뷸라이저)만 ordered → **좌측 윤서아 1명("미수행 1")**. 클릭 대상=윤서아.
- 번호(권장 4): ①수행 대기 처치(헤더) `getByRole("heading",{name:"수행 대기 처치"})` ②환자 행 `getByRole("button").filter({hasText:"윤서아"})`(또는 `getByText("미수행 1")`) ③처치기록 입력 `getByPlaceholder("처치기록 내용(선택 — 주민번호 등 민감정보 금지)")` ④수행 버튼 `getByRole("button",{name:"수행"})`.
- flow: 의사 지시 처치가 여기 뜸 → 환자 선택 → (처치기록 선택 입력) → "수행" → performed. **이미 수행된 처치는 "수행 완료" 잠금 카드로 바뀌어 다시 수행 못 함(재수행 차단·3중: 버튼 disable·완료 카드·서버 409).** 간호사는 진료 허브 권한이 없어 이 워크리스트가 처치 진입점.

#### 2. 활력징후 입력 `/nurse/vitals` — `vitals-page.tsx`(+`vitals-input-form.tsx`)

- 번호(권장 5): ①오늘 활성 내원(헤더) `getByRole("heading",{name:"오늘 활성 내원"})` ②환자 행 `filter({hasText:"오세훈"})` ③활력 입력(대표=수축기 혈압) `getByLabel("수축기 혈압 (mmHg)")` ④활력징후 메모 `getByLabel("활력징후 메모")` ⑤활력징후 기록 버튼 `getByRole("button",{name:"활력징후 기록"})`. (선택 ⑥최근 활력 `getByRole("heading",{name:"최근 활력"})`.)
- 6항목 라벨(aria): 수축기 혈압 (mmHg)·이완기 혈압 (mmHg)·맥박 (bpm)·체온 (°C)·호흡수 (/min)·SpO₂ (%). flow: 환자 선택 → 측정한 항목만 입력(최소 1개·비정상은 빨강 강조) → "활력징후 기록". 우측 "최근 활력"이 직전 측정값 표시.
