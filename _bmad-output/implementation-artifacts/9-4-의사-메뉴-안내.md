# Story 9.4: 의사(doctor) 메뉴 안내

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 의사,
I want 내 진료 화면(진료 대기·진료 허브·판독·환자 검색)의 사용법을 실제 화면·번호 하이라이트·단계 설명으로 보기를,
so that 진료 시작→SOAP·진단·오더→(수납에서 완료) 흐름과 본인 진료과 고정·주민번호 reveal 감사를 정확히 이해하고 진료한다.

## Acceptance Criteria

1. **의사 메뉴 전수 수록.** 의사 계정으로 `/help`에 들어가면 사이드바에 보이는 **3개 메뉴 전부**가 안내로 제공된다: **진료 대기 `/doctor/waiting`**(진료 대기 + 진료 허브 2화면)·**판독 `/doctor/radiology`**·**환자 검색 `/patients`**. "준비 중" 플레이스홀더로 남는 의사 메뉴가 없어야 한다.

2. **(이 스토리의 신규 작업) 판독 메뉴 안내.** `/doctor/radiology`(판독 워크리스트) 가이드가 신설된다 — 실제 캡처 화면 PNG + 핵심 동작요소 번호 하이라이트(①②③④) + 번호별 설명표(`hotspots`) + 작업 흐름(`flow`). 좌측 판독 대기 목록 → 우측 판독 패널(영상·소견·결론·완료)의 사용법이 담긴다. 캡처 스펙의 번호(n) = help-content `hotspots[].num` 1:1.

3. **진료 흐름·정책 설명(AC And 절).** 진료 대기/허브 가이드에 **진료 시작 → SOAP·진단·오더 → (원무 수납에서 완료)** 흐름, **본인 진료과 고정**, **주민번호 reveal = 감사 기록**이 설명되어 있다(9.1 시범 콘텐츠가 이미 충족 — 누락 시 보강). 판독 가이드 flow에는 **촬영 수행 → 판독(소견·결론) → 검사 오더 완료** 흐름이 담긴다.

4. **(9.2 파이프라인 소비) 캡처 자동 생성·정적 임베드.** 판독 화면 이미지는 `tools/screenshots/specs.mjs`에 doctor radiology 스펙을 추가하고 `node capture.mjs doctor radiology`로 생성해 **`web/public/help/doctor/radiology.png`**에 배치한다(런타임 캡처 0·외부 편집 도구 0). 캡처는 더미 데이터(**판독 대기 영상검사 = 활성 내원의 performed imaging**)가 적용된 환경 대상. 의사 진료 대기·허브(9.1) 이미지는 무변경.

5. **회귀 무파괴·범위 한정.** 변경 = `web/src/lib/help/help-content.ts`(판독 가이드 추가)·`tools/screenshots/specs.mjs`(doctor radiology 스펙)·`web/public/help/doctor/radiology.png`(생성)·테스트(`help-content.test.ts`·`help-guide.test.tsx` 의사 커버리지 갱신). **앱 화면 코드·API·마이그·권한·생성 TS 무변경.** `npm run build`/lint/타입체크 통과. 기존 도움말 셸·원무(9.3)·의사 시범(9.1)·사이드바 회귀 0.

## Dev Notes

### 컨텍스트 요약 — 무엇을 만드나
9.4는 의사 역할 콘텐츠를 **완성**한다. 의사 3메뉴 중 진료 대기/허브(9.1 시범)·환자 검색(9.3 공유 `/patients`)은 이미 있고, **판독 `/doctor/radiology`만 비어 있다**(플레이스홀더). 9.4 = 판독 가이드 1개 신설 + 캡처 1장(`doctor/radiology.png`) + 의사 콘텐츠 완성으로 깨지는 테스트 1개 갱신. 9.3에서 확립한 패턴(specs goto + help-content + 캡처 + 테스트)을 그대로 적용 — **작고 집중된 스토리**.

### ⚠️ 비자명·실수 방지
