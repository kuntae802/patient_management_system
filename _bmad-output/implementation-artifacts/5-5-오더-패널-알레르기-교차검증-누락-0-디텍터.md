# Story 5.5: 오더 패널 · 알레르기 교차검증 · 누락 0 디텍터

Status: review

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 의사,
I want 오더 패널에서 알레르기·중복·미수행 위험을 시스템이 잡아 주기를,
so that 환자 안전이 절차로 강제된다.

## Acceptance Criteria

(epics.md Epic 5 / Story 5.5 — UX-DR13 order-panel · UX-DR21 Clinical Safety Patterns②⑥. **Epic 5 통합 정점 스토리** — 5.2 처방·5.3 검사·영상·5.4 처치 패널을 UX-DR13 탭 패널로 통합하고 안전 레이어를 얹는다.)

1. **(UX-DR13) 오더 패널 통합 — 탭·추적 라인·pay-chip·수가 프리뷰** — 진료 허브 우 오더 pane이 **처방/검사/영상/처치 탭**(카운트 배지)으로 재구성된다. 각 오더 아이템은 **추적 라인**(지시자 이름·시각 + 수행자 이름 또는 "수행 대기")과 **급여/비급여 pay-chip**(마스터 `coverage_type`)을 표시한다. 추가는 **마스터 검색 피커로만**(free-text 차단 — 기존 유지). 패널에 **수가 자동 산정 프리뷰**("자동 산정" 마커 + 급여/비급여 소계)가 표시된다. 프리뷰·pay-chip은 **표시 전용** — 실제 수가 자동발생(수납상세 적재)·본인부담 산정은 **5.10/Epic 7**.
2. **(UX-DR21②) 알레르기↔오더 교차검증 — 서버 강제 사유 오버라이드+감사** — 처방 발행 시 **서버**가 환자 기록 알레르기(`patients.allergies` 자유텍스트)와 약품명/성분을 토큰 대조하고, 활성 투약 동일성분(FR-052)을 패널에 surface한다. 알레르기 매칭 시 **오버라이드 사유 미입력이면 409 `allergy_conflict`로 차단**, **사유 입력 시 발행 진행 + 사유가 `prescription_details.allergy_override_reason`에 기록되어 감사 트리거로 append-only 캡처**된다. 클라는 발행 전 동일 휴리스틱으로 **즉시 경고 + 사유 입력 필드**를 인라인 노출한다(서버는 권위·재검증).
3. **(UX-DR21⑥) 누락 0 디텍터** — 지시(`ordered`/`issued`) 상태로 **임계치(기본 30분) 초과**한 미수행 오더가 진료 허브 오더 패널에 **가시 인디케이터**(색+글리프+라벨 3중 인코딩, 음영 비의존 UX-DR20)로 surface되어 누락 0이 보장된다. (직역별 워크리스트 측 인디케이터는 워크리스트 UI 구축 스토리 5.7 간호·5.8 방사선에서 통합 — 본 스토리는 워크리스트를 만들지 않음, §스코프 경계.)

- fee_schedules(seed.sql:341-360·18종)·drugs(seed.sql:364-382·17종) VALUES에 `coverage_type` 추가. **⚠️ `on conflict (lower(code)) do nothing`** → 비-reset 재실행 시 기존 행 미갱신. dev 표준 = `supabase db reset`(fresh)이라 reset 시 반영(6-3 "reset && pytest"). 컬럼 순서 = `(code, name, amount_krw, category, coverage_type, effective_from, effective_to)` 형태로 VALUES 전 행 갱신.
