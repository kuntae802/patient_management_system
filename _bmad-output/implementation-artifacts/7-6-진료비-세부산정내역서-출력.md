## Story

As a 원무 직원,
I want finalized 수납 건에 대해 「진료비 세부산정내역서」를 미리보기하고 브라우저 인쇄(`Ctrl P`)/PDF로 출력하기를,
so that 환자가 라인별 산정 근거(항목분류·일자·코드·명칭·단가·횟수·일수·금액·본인부담·공단부담)를 받고, 영수증과 동일한 인쇄=감사·주민번호 마스킹·파일명 PII 금지 정책이 적용된다.

> **Epic 7 여섯 번째 스토리 — 표준 문서 출력 레이어의 두 번째 문서(7.5 영수증에 이은 세부산정내역서).** 7.5(`0049`)가 finalized 수납 건을 법정 서식 「진료비 계산서·영수증」으로 렌더하며 **세 가지 인프라를 이미 구축했다**: ① `clinic_profile`(요양기관 마스터) ② `log_payment_document_export(p_encounter_id, p_document_type)` **제네릭** 내보내기 감사 RPC(0049·`document_type` 파라미터 보유) ③ `GET .../payment/receipt` → `ReceiptResponse`(요양기관·환자 masked RRN·진료·금액·**`details` 라인 전 컬럼**)와 `@media print`(`.receipt-paper`) Batang serif. **세부산정내역서는 그 ReceiptResponse 와 동일 데이터의 "다른 렌더링"**(대분류 집계 → 라인별 전개)이다. 따라서 **7.6 = 본질적으로 web 렌더링 스토리**: 마이그레이션 0 · API 1줄(`DocumentExportRequest` Literal `['receipt']`→`['receipt','statement']`) · 신규 web 컴포넌트 `statement-document.tsx`(라인별 표) + `billing-detail.tsx` 미리보기를 **문서 탭 토글**(목업 key-billing.html L661-664)로 전환. **신규 라이브러리 0 · 신규 권한 0 · 신규 엔드포인트 0.**

### ⚠️ 인쇄=감사 = 활성 탭 기준 (stale closure 주의)

7.5 beforeprint 리스너는 `exportReceipt(encounterId, "receipt")` 하드코딩. 7.6 은 탭 토글이므로 **활성 탭 문서가 감사돼야** 한다:
```tsx
useEffect(() => {
  if (!receipt) return;
  const onBeforePrint = () => { void exportReceipt(encounterId, activeDoc).catch(() => {}); };
  const prevTitle = document.title;
  document.title = activeDoc === "statement" ? `세부내역서_${receipt.patient.chart_no}` : `영수증_${receipt.patient.chart_no}`;
  window.addEventListener("beforeprint", onBeforePrint);
  return () => { window.removeEventListener("beforeprint", onBeforePrint); document.title = prevTitle; };
}, [receipt, encounterId, activeDoc]);  // ← activeDoc 의존성 필수(stale 'receipt' 차단)
```
- `activeDoc` 의존성 누락 시 탭 전환 후에도 'receipt' 감사 → granularity 깨짐.
- 인쇄는 활성 탭 `.receipt-paper` 만 출력 — **비활성 탭 문서는 DOM 미렌더**(조건부 렌더 `activeDoc === 'receipt' ? <Receipt/> : <Statement/>`)해야 print 에 1개만 잡힌다(둘 다 렌더 시 인쇄에 양쪽 출력).
