# Story 6.3: 예약 캘린더 · 더블부킹 차단

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a **원무 직원**,
I want **캘린더에서 슬롯을 보고 예약하되 더블부킹이 차단되기를**,
so that **예약 충돌이 0이 된다.**

## Acceptance Criteria

**[AC1] 예약 캘린더 — 시간레일×의사 열·일 보기·슬롯 상태 렌더 (음영 비의존)**
- **Given** 예약 캘린더(UX-DR15: 시간레일(세로 30분) × 열=의사, **일(Day) 보기**)에서
- **When** 슬롯 상태(가능/확정/완료/노쇼/취소/휴진/점심/지남)를 렌더하면
- **Then** 각 슬롯이 **채움+테두리+패턴**(휴진=빗금 PATTERN·취소=취소선·확정=환자명+◐ 등)으로 표시된다(음영·색 단독 의존 금지, UX-DR20). 점심은 전 열을 가로지르는 band("점심" · 예약 불가).

**[AC2] booking-peek 슬라이드오버 — 빈 슬롯 클릭 → 예약 생성**
- **Given** 빈(가능) 슬롯을 클릭할 때
- **When** booking-peek 슬라이드오버가 열리면
- **Then** **진료과·담당의·환자검색(이름·차트·연락처)·날짜/시간·메모·예약 확정 SMS 체크·저장/취소**가 제공되고, 클릭한 슬롯의 의사·날짜·시간이 프리필된다. 저장 시 예약(`status='booked'`)이 생성되고 캘린더에 확정 슬롯으로 반영된다.

**[AC3] 더블부킹 인라인 차단 (FR-013)**
- **Given** 동일 의사·동일 시간대에 이미 활성(`booked`) 예약이 있을 때
- **When** booking-peek 에서 저장하면
- **Then** 서버 **409**(`double_booking`)가 인라인 경고 칩으로 표시되고 **저장되지 않는다**(슬라이드오버 유지). DB EXCLUDE(0031)가 최종 불변식.

> **범위 노트(에픽 6.3):** 6.3 = 예약 **생성**(쓰기)·캘린더 일 보기·booking-peek·더블부킹 409 표면화. **변경·취소·노쇼 전이·예약 상태머신 트리거·`reservation_id`→내원 링크·대기 흐름 반영·도착 접수(check-in) = 6.4**(원무 대리 생성·변경·취소). 6.3 은 INSERT(booked)만 — 노쇼/취소/완료 상태는 캘린더가 **렌더**하되 실제 전이는 6.4 가 만든다. 6.2 의 슬롯 계산(`compute_available_slots`)·`SLOT_STATUS_META`·`appointment.read`·`bookable-doctors` 를 재사용한다.
