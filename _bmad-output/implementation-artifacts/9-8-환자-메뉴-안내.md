# Story 9.8: 환자(portal) 메뉴 안내

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 환자,
I want 내 포털 화면(예약·내 진료기록·처방/검사결과·수납/영수증)의 사용법을 실제 화면·번호 하이라이트·단계 설명으로 보기를,
so that 예약·내 기록을 선배나 직원의 도움 없이 스스로 확인한다.

## Acceptance Criteria

1. **환자 도움말 진입·전용 페이지.** 환자 포털 마이(`/portal`)의 계정 동작 섹션에 "도움말" 링크가 보이고, 누르면 환자 전용 도움말 페이지(`/portal/help`)로 이동한다. 이 페이지는 `(patient)` 셸에 속해 인증이 필요하고(직원 계정은 `/home`으로 차단), 직원용 `/help`와 별개다(기존 직원 `/help`·HelpGuide·STAFF_NAV 무변경).

2. **(FR-252·253) 환자 메뉴 전수 수록 = 스크린샷 + 번호 하이라이트 + 설명표 + 흐름.** 도움말 페이지에 환자 4개 메뉴가 빠짐없이 안내된다: **예약 `/booking`·내 진료기록 `/records`·처방/검사결과(`/records` 내원 펼침)·수납/영수증(`/portal` 수납 카드 + `/receipts/[id]` 영수증)**. 각 화면 = 캡처 PNG + 번호 하이라이트(①②③…) + 설명표 + (선택)흐름. 캡처 스펙 번호(n) = 콘텐츠 `hotspots[].num` **1:1**. "준비 중" 플레이스홀더로 남는 환자 메뉴가 없어야 한다.

3. **(AC And 절) 본인 데이터만(RLS) 보임 설명.** 환자 포털은 로그인 본인의 데이터만 보인다(self-read·세션 uid 스코프). 이 점이 도움말의 intro/hotspot/flow에 명시된다(각 화면의 "신뢰 노트"=ShieldCheck "본인 정보만 안전하게 표시" 안내를 가리키거나 서술). 미연결 환자는 "본인 진료기록 연결"부터 한다는 안내도 포함된다.

4. **(9.2 파이프라인 소비·모바일 캡처) 캡처 자동 생성·정적 임베드.** 환자 화면 이미지는 `tools/screenshots/specs.mjs`에 patient 스펙을 추가하고 **모바일 뷰포트**로 생성해 **`web/public/help/patient/<screen>.png`**에 배치한다(런타임 캡처 0·외부 편집 도구 0). 환자 셸은 모바일 폭(`max-w-md`)이므로 캡처는 폰 뷰포트(예: 390×844)로 한다(직원 1440×900과 다름). 캡처 대상 = 클라우드 + **연결된 데모 환자** 환경. 직원 역할(9.3~9.7) 콘텐츠·이미지는 무변경.

5. **회귀 무파괴·범위 한정.** 신규 = `web/src/app/(patient)/portal/help/page.tsx`(라우트)·`web/src/components/portal/patient-help-guide.tsx`(환자 렌더러)·`web/src/lib/help/patient-help-content.ts`(콘텐츠)·`web/public/help/patient/*.png`·테스트. 수정 = `web/src/app/(patient)/portal/page.tsx`(도움말 링크 1줄)·`tools/screenshots/specs.mjs`(patient 스펙)·`tools/screenshots/capture.mjs`(계정별 비밀번호 지원·아래 비자명 #6). **앱 임상/수납 기능 코드·API·마이그·권한·생성 TS 무변경.** `npm run build`/lint/타입체크 통과. 기존 직원 도움말(9.1~9.7)·환자 포털 화면·사이드바·탭바 회귀 0.

9.8은 **Epic 9 마지막 스토리**이자 **유일하게 환자(`(patient)`) 셸을 다루는** 도움말 스토리. 직원 역할(9.3~9.7)은 기존 `/help`·HelpGuide·STAFF_NAV에 데이터+캡처만 추가했지만, **환자 포털은 직원 셸과 완전히 분리**(layout.tsx·PATIENT_NAV 없음·PatientTabBar 3탭 로컬 렌더·기존 `/help`는 직원 전용으로 비직원 차단)되어 **환자 전용 도움말 경험을 새로 만든다**. 9.1이 명시적으로 "환자 포털 도움말 = 9.8"로 이월했다(9.1~9.7은 환자 셸 무변경).

**확정 설계(사용자 결정):** 새 라우트 `/portal/help`(`(patient)` 셸) + 마이(`/portal`) 계정 동작에 "도움말" 링크 + 환자 전용 렌더러(filterNav 없음·고정 4메뉴) + 콘텐츠는 직원 `HelpScreen`/`HelpHotspot` 타입 재사용 + 모바일 뷰포트(390×844) 캡처 → `web/public/help/patient/`. 기존 직원 `/help`·HelpGuide·STAFF_NAV·셸 전부 무변경(회귀 위험 최소).

### ⚠️ 비자명·실수 방지
1. **🔴 환자 도움말 ≠ 직원 도움말 재사용.** 기존 `(staff)/help`는 `requireStaff()`로 비직원을 `/portal`로 보낸다 — 환자는 못 본다. route group `(staff)`/`(patient)`가 둘 다 `/help`를 정의할 수 없으므로(URL 충돌) **새 경로 `/portal/help`**. HelpGuide 컴포넌트는 `usePermissions`+`filterNav(STAFF_NAV)`라 환자엔 무용 — **환자 전용 렌더러 신설**(고정 4메뉴·권한 필터 없음). 단 데이터 모델(`HelpScreen`/`HelpHotspot`)·`helpImageSrc`·시각 패턴(이미지+번호표+flow)은 재사용.
2. **🔴 모바일 단일 컬럼 렌더.** 환자 포털은 `max-w-md`(폰). 직원 `HelpScreenBlock`은 `lg:grid-cols-[2fr_1fr]`(이미지 좌·설명 우 2컬럼)인데 폰 폭엔 안 맞다 → 환자 렌더러는 **단일 컬럼**(이미지 `w-full` 위 · 번호 `<ol>` 아래 · flow 아래). 캡처도 폰 뷰포트라 이미지 자체가 세로로 길다.
3. **🔴 모바일 뷰포트 캡처(env 오버라이드).** `lib.mjs` `VIEWPORT={width:PMS_VW||1440, height:PMS_VH||900}` — 전역. 환자 캡처는 `PMS_VW=390 PMS_VH=844 node capture.mjs patient`로 폰 뷰포트. (파이프라인 코드 수정 없이 env로 해결. 직원 캡처는 기존대로 1440×900.) 폰 폭이라 포털 콘텐츠가 화면을 꽉 채워 스크린샷 가독성↑(1440이면 가운데 좁은 컬럼+여백).
4. **🔴 캡처 중 mutation 금지(9.7 교훈 계승).** 환자 포털의 유일 쓰기=예약 확정(POST /appointments). **booking goto는 진료과/의사/날짜 선택까지만**(슬롯 fetch=GET·읽기)·**"예약 확정하기" 클릭 금지**(실 예약 생성→데모 DB 오염). records 펼침(GET detail)·수납 카드 클릭(네비)·영수증 로드는 전부 읽기(safe). annotate는 boundingBox만.
5. **🔴 self-read 데이터 선행(캡처 최대 리스크).** 환자 포털은 **본인 데이터만**(세션 uid→`get_current_patient`). 데모 환자가 **patient 레코드에 연결+데이터 보유**해야 4화면이 비지 않는다. ⚠️ demo_seed에 데모 auth 환자·연결 **없음**(Epic 8에서 클라우드 수동). 캡처 전: ①데모 환자 로그인 프로브 ②미연결이면 `/onboarding`에서 demo_seed 환자(완료 내원+처방/검사+finalized 수납 보유)의 이름+주민번호로 self-link ③예약 가능 진료과·완료 내원·finalized 수납 노출 확인. (연결이 곧 mutation이나 이는 캡처 환경 준비로 1회·예약 확정과 다름.)
6. **🔴 캡처 러너 계정별 비밀번호.** 데모 환자 비번=`Patient1234!`(직원 공통 `Staff1234`와 다름). `capture.mjs`는 `login(page, account, PASSWORD)` 단일 — 환자 로그인 실패. **`login(page, account, spec.password ?? PASSWORD)`로 보강** + patient 스펙에 `password:"Patient1234!"`. 직원 스펙은 password 미지정→PASSWORD 폴백(무영향). 이 한 줄이 9.8의 유일한 파이프라인 코드 수정.
7. **번호 가림 주의(하단 고정 요소).** records/portal은 PatientTabBar(`fixed bottom-0`·~56px), booking은 sticky CTA(`fixed bottom-0`)가 하단을 덮는다. 번호 hotspot은 **이들에 가리지 않는 영역**으로(boundingBox가 폴드 안). 폰 뷰포트(844 높이)에서 상단부 요소 위주.
8. **RLS 설명(AC#3).** 각 화면의 "신뢰 노트"(ShieldCheck "본인 정보만 안전하게 표시됩니다. 다른 사람은 볼 수 없어요.")가 RLS의 가시적 증거 — hotspot/intro로 가리키거나 서술. 미연결 환자는 "본인 진료기록 연결"부터(intro/flow).

- **hotspot(권장 4~5):** ①진료과 ②의사 ③날짜 칩 ④시간 슬롯 (⑤예약 확정하기 — 버튼 위치만·**클릭 금지**). 빈 시간="이 날짜에 예약 가능한 시간이 없어요."

- **flow:** 내원을 골라 펼치면 처방·검사 결과. 본인 것만 보임(신뢰 노트).

- **hotspot(권장 3~4):** ①처방받은 약(약명·용량·일수) ②검사 결과 요약(검사명·요약) ③정상/주의 배지 (④쉬운 말 요약 — 8.1/8.2 patient_friendly_note/patient_result_summary).
- **goto:** `/records` → 완료 내원 카드 "처방·검사 결과 보기" 클릭(GET detail·읽기) → 노출.
- **flow:** 완료된 진료의 처방·검사 결과를 쉬운 말 요약으로. 수치/영상 원본은 범위 밖(의료진 상담).

#### D. 수납/영수증 `/portal`(수납) + `/receipts/[id]`(영수증) (`payment-history.tsx`·`receipt-detail.tsx`)

- **영수증(`/receipts/[id]`)**: "마이로" 뒤로. h1 "진료비 영수증". 진료 항목 대분류 표·합계(총 진료비/건강보험 부담/**내가 낸 금액** 강조)·인쇄 버튼 "영수증 인쇄·저장"(화면=친화요약·인쇄=법정서식). 404="영수증을 찾을 수 없어요."
- **hotspot(권장 화면당 3):** 마이=①수납 카드(날짜·납부액) ②완료 배지 ③신뢰 노트. 영수증=①항목 대분류 표 ②내가 낸 금액(강조) ③인쇄 버튼.

- **flow:** 마이에서 수납 카드를 골라 영수증을 본다. 친화 요약(화면) + 법정 서식(인쇄). 본인 finalized만.
