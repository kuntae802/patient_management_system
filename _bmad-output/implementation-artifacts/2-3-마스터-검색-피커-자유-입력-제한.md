# Story 2.3: 마스터 검색 피커 · 자유 입력 제한

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 임상·원무 직원,
I want 진단(KCD)·약품·수가를 **재사용 검색 피커로만** 선택하기를,
so that 비표준 자유 입력(free-text) 없이 단일 진실(마스터)이 강제되고, Epic 4(진단)·Epic 5(약품·수가)가 동일 컴포넌트를 그대로 소비한다.

## Acceptance Criteria

**AC1 (FR-202) — 마스터 검색·선택 강제 · free-text 차단**
**Given** 마스터(diagnoses·drugs·fee_schedules)가 존재하는 입력 지점에서
**When** 재사용 검색 피커(`MasterSearchPicker kind="diagnosis"|"drug"|"fee_schedule"`)를 열고 코드 또는 명칭으로 검색하면
**Then** 마스터에서 **검색·선택만** 가능하고, 목록에 없는 임의 텍스트는 **값으로 커밋되지 않는다**(자유 입력 차단). 선택된 결과는 `onValueChange`(또는 `onSelect`)로 소비처에 전달된다. 노출 후보는 **"현재 유효" 코드만**(`is_active=true AND effective_from ≤ today AND (effective_to IS NULL OR effective_to ≥ today)`).

**AC2 (UX-DR19·UX-DR20) — 키보드 완전 조작 + aria-live 결과 안내**
**Given** 검색 피커가 열린 상태에서
**When** 키보드로 검색(타이핑)·이동(↑/↓)·선택(Enter)·닫기(Esc)하면
**Then** 마우스 없이 완전 조작되고, 결과 개수 변화가 `aria-live`(Base UI `Combobox.Status`) 리전으로 스크린리더에 안내되며, `role=combobox/listbox/option`·`aria-activedescendant`·`:focus-visible` 포커스 링이 보장된다. 결과 없음 상태(`Combobox.Empty`)가 명시된다. 색만/음영만으로 의미를 싣지 않는다(색+글리프/라벨).

**AC3 — Epic 4·5 재사용 계약(forward-compatible 인터페이스)**
**Given** 동일 피커 컴포넌트가
**When** Epic 4.7(진단 부착)·5.2(처방)·5.5/7.2(수가)에서 소비될 때
**Then** `kind` prop로 3종 마스터를 전환하고, `multiple` prop로 단일(약품·수가)/다중(진단 주·부상병) 선택을 지원하며, 선택 결과가 소비처가 필요로 하는 식별 필드(진단=`{code,name}`, 약품=`{code,name,ingredient_code}`, 수가=`{code,name,category,amount_krw}`)를 담아 전달된다. **이 스토리는 컴포넌트와 테스트를 산출하고, 임상 입력 화면(Epic 4/5)은 만들지 않는다.**

> **AC 해석(중요·dev agent 필독):** 이 스토리는 **프론트엔드 전용**이다. **DB 마이그레이션·FastAPI 엔드포인트·새 권한·새 nav 항목은 추가하지 않는다.** Story 2.2가 깐 자산을 소비한다: ① `0007`의 diagnoses·drugs·fee_schedules 테이블(RLS authenticated SELECT 이미 GRANT) → web Supabase 직접조회로 읽음, ② `lib/admin/masters.ts`의 타입·`isCurrentlyValid`·`formatKrw` 재사용. **핵심은 `@base-ui/react@1.6.0`의 `Combobox` 프리미티브 위에 얇은 재사용 래퍼를 만드는 일**(자유 텍스트를 허용하는 `Autocomplete`가 아니라 목록 강제 `Combobox`). 콤보박스·키보드·ARIA·포커스 트랩을 **직접 구현하지 말 것** — Base UI가 네이티브 제공. shadcn `cmdk`/`Command`·downshift·react-select 등 **새 라이브러리 도입 금지**.

### ⏱️ "현재 유효" = DB 권위로 통일 (2.2 이월 항목 해소)

[Source: deferred-work.md:9-10, web/src/lib/admin/masters.ts:141-170]

2.2 코드리뷰가 2.3로 배정한 이월: 관리화면 배지는 `todayISO()`=**브라우저 로컬 시간**으로 계산 → 피커가 다른 today(예 DB)를 쓰면 자정 경계·비-KST 브라우저에서 배지("유효")와 피커(제외)가 불일치.

**해소 방침(가장 단순·정확):**
1. **피커 `today`는 서버 주입 필수 prop**(브라우저 시계 기본값 제거). 단일 진실 = 서버 today(KST). `isCurrentlyValid(row, today)`는 이미 today 인자를 받도록 설계됨(masters.ts:164) — 그대로 활용.
2. **관리화면도 같은 서버 today를 RSC에서 주입**(Task 3) → 배지·피커가 동일 today 공유. `MastersManager`는 폴백으로 기존 클라 today 유지(회귀 0).
3. 대안(채택 안 함): RPC/뷰의 `current_date` SQL 필터를 단일 진실로 — 더 무겁고 현재 불요. 후속 승격 후보로 남김.

서버 today 계산: RSC(서버 컴포넌트)에서 `todayISO()` 호출 시 **서버 시간(KST 컨테이너)** 사용 → DB(Postgres)와 같은 날짜 경계. (배포: TZ=KST.)
