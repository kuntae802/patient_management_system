# Story 9.6: 방사선사(radiologist) 메뉴 안내

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 방사선사,
I want 내 영상 업무 화면(촬영 워크리스트·영상 업로드·장비 관리)의 사용법을 실제 화면·번호 하이라이트·단계 설명으로 보기를,
so that 촬영 수행 → 영상 업로드 → 의사 판독 흐름을 끊김 없이 수행한다.

## Acceptance Criteria

1. **방사선사 메뉴 전수 수록.** 방사선사 계정으로 `/help`에 들어가면 사이드바에 보이는 **3개 메뉴 전부**가 안내로 제공된다: **촬영 워크리스트 `/radiology/worklist`·영상 업로드 `/radiology/upload`·장비 관리 `/radiology/equipment`**. "준비 중" 플레이스홀더로 남는 방사선사 메뉴가 없어야 한다.

2. **(FR-252) 각 메뉴 = 실제 화면 스크린샷 + 번호 하이라이트 + 설명표 + 흐름.** 3개 메뉴 각각 캡처 PNG + 번호 하이라이트(①②③…) + `hotspots` 설명표 + `flow`. 캡처 스펙 번호(n) = help-content `hotspots[].num` 1:1. **⚠️ 촬영 워크리스트와 영상 업로드는 동일 컴포넌트(`RadiologyWorklistPage`)를 렌더하는 같은 작업 공간**(h1만 다름) — 두 가이드는 같은 화면을 (촬영 워크리스트=목록·수행 중심 / 영상 업로드=영상 파일 선택·장비 배정 중심)으로 **강조점을 달리해** 안내한다(번호가 이미지에 구워지므로 각각 별도 캡처).

3. **핵심 흐름 설명(AC And 절).** **촬영 수행 → 영상 업로드 → 의사 판독** 흐름이 flow/intro에 명시된다: 좌측 촬영 대기 검사를 선택해 영상을 1장 이상 업로드하고 장비를 배정한 뒤 "촬영 수행"하면, 그 검사가 의사의 판독 워크리스트(9.4 `/doctor/radiology`)로 넘어간다.

4. **(9.2 파이프라인 소비) 캡처 자동 생성·정적 임베드.** 3개 화면 이미지는 `tools/screenshots/specs.mjs`에 radiologist 스펙 3개를 추가하고 `node capture.mjs radiologist`로 생성해 **`web/public/help/radiologist/<screen>.png`**에 배치한다(런타임 캡처 0·외부 편집 도구 0). 캡처는 더미 데이터 + radiologist 권한 환경 대상. 원무(9.3)·의사(9.1/9.4)·간호(9.5) 콘텐츠·이미지는 무변경.

5. **회귀 무파괴·범위 한정.** 변경 = `web/src/lib/help/help-content.ts`(radiologist 가이드 3개)·`tools/screenshots/specs.mjs`(스펙 3개)·`web/public/help/radiologist/*.png`(생성)·테스트(`help-content.test.ts` 커버리지 + 선택 `help-guide.test.tsx` 렌더). **앱 화면 코드·API·마이그·권한·생성 TS 무변경.** `npm run build`/lint/타입체크 통과. 기존 도움말 셸·원무/의사/간호 콘텐츠·사이드바 회귀 0.

> **범위 경계 (반드시 지킬 것):** 이 스토리는 **방사선사(radiologist) 역할 콘텐츠 완성**까지다.
> - 9.6 = radiologist 3메뉴 help-content 가이드 + 캡처 스펙 3개 + PNG 3장 + (필요 시) 테스트.
