# Story 7.7: 원외처방전 출력 · 발급

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 원무 직원,
I want 발행된(`issued`) 처방에 대해 표준 「원외처방전」을 미리보기·브라우저 인쇄(`Ctrl P`)/PDF로 출력하고, 명시적 "발급 확정"으로 처방을 `issued→dispensed` 전이하기를,
so that 환자가 원외 약국에서 조제받고, 발급 이력이 감사로 남으며, 7.5/7.6과 동일한 인쇄=감사·주민번호 마스킹·파일명 PII 금지 정책이 처방전에도 적용된다.

> **Epic 7 일곱 번째 스토리 — 표준 문서 출력 레이어의 세 번째 문서(7.5 영수증·7.6 세부내역서에 이은 원외처방전).** 7.5/7.6과 달리 **수납(payment) 문서가 아니라 처방(prescription) 문서**다. 인쇄 인프라(`.receipt-paper`·`@media print`·`font-legal-serif`·`formatKstDate`·`Won`)는 재사용하나, 데이터 출처(payment→prescription)·감사 스코프(payments→prescriptions)·생명주기(finalize→dispense)가 **근본적으로 다르므로** 신규 표면이 7.6(마이그 0)보다 크다.
>
> **Story 5.1이 명시 이월한 작업**: `dispense_prescription` RPC + `prescription.dispense` 권한 = "Epic 7(7.7 원외처방전)"(5-1 스토리 L115·L238). 처방 상태머신(`issued→dispensed` CHECK·`enforce_prescription_transition` 트리거·`dispensed_at` 컬럼)은 **0015에서 이미 깔렸으나**(0015:44-48·140) 발급 경로(권한 게이트·RPC)가 없어 현재 `service_role` 직접 UPDATE로만 도달 가능. 7.7이 그 경로를 정석으로 완성한다.
>
> ⚠️ **설계 결정 4건(사용자 확정 2026-06-24·AskUserQuestion)** — Dev Notes §경계·설계 결정 참조. ① **발급 모델 = 명시적 "발급 확정" 버튼**(인쇄=감사·자유 / 발급=`dispense_prescription` RPC로 `issued→dispensed`·비가역 1방향 전이·분리·7.4 finalize 패턴 계승) ② **발급 권한 = `prescription.dispense` 신규 권한**(reception+admin grant·문서 읽기 엔드포인트도 동일 게이트·admin 부트 grant 재실행) ③ **진입 화면 = 수납 화면(billing-detail) 내 처방전 섹션**(finalize 무관·발행 처방만 있으면 노출) ④ **발급자 기록 = 감사로그 actor만**(`dispensed_by` 컬럼 미추가·`trg_prescriptions_audit`가 UPDATE를 actor와 자동 기록·`dispensed_at`만).

**AC2 — `dispense_prescription` RPC (issued→dispensed·perform_examination 템플릿·FR-080):**
- `dispense_prescription(p_prescription_id uuid) returns public.prescriptions` — `language plpgsql security definer set search_path = public`. **0015 `perform_examination`(L194-219) 정확히 미러**:
  - `if not public.has_permission('prescription.dispense')` → `raise ... using errcode = 'insufficient_privilege'`(방어심층·라우터 게이트 1차선·동일-txn 재평가).
  - `select * into v_row from public.prescriptions where id = p_prescription_id for update`. not found → `PT404`.
  - **소스 상태 선검사**: `if v_row.status <> 'issued'` → `PT409`(이미 dispensed 재발급 차단·일방향). 메시지 `'invalid prescription transition: % -> dispensed'`.
  - `update public.prescriptions set status = 'dispensed', dispensed_at = now(), updated_at = now() where id = p_prescription_id returning * into v_row`. **`dispensed_by` SET 없음**(설계 결정 ④ — 컬럼 미추가).
  - return v_row.
- **EXECUTE posture**: `revoke all ... from public, anon, authenticated; grant execute ... to authenticated, service_role`(0015:310-312 perform_* 동형 — RPC가 has_permission 자체 게이트라 authenticated 직접 호출도 안전·방어심층).
- **발급자 감사 = 자동**: `trg_prescriptions_audit`(0015:398-400·`audit_trigger_fn` 0004)가 이 UPDATE를 actor(`app.actor_id`→`auth.uid()`)와 함께 `audit_logs`에 append-only 기록. ⚠️ FastAPI가 RPC 호출 전 커넥션에 `app.actor_id`(jwt sub)를 세팅하는 기존 `_run_authed` 경로를 타야 actor가 정확히 잡힌다(perform_examination 호출 패턴 동일).

## Dev Notes

### 스토리 성격 — Epic 7 문서 출력 레이어의 세 번째 문서(처방 스코프·payment 아님)

7.5(0049 영수증)·7.6(마이그 0 세부내역서)는 **payment 스코프**(finalized 수납 건). 7.7은 **prescription 스코프** — 처방(prescriptions 0015)을 출력하고 발급(issued→dispensed) 전이한다. 인쇄 인프라(`.receipt-paper`·`@media print`·`font-legal-serif`·`formatKstDate`·`Won`)는 재사용하나 **데이터·감사·생명주기가 payment와 무관**하므로 신규 표면이 7.6보다 크다:
- 마이그 0050: `dispense_prescription` RPC·`prescription.dispense` 권한·`log_prescription_document_export` RPC(5.1 이월 청산)
- API: `fetch_prescription_document` + 3 엔드포인트(document GET·dispense POST·export POST)
- Web: `prescription-document.tsx`(신규)·billing-detail 처방전 섹션·lib 데이터 계층
- ❌ 신규 라이브러리 · ✅ 신규 권한 1(prescription.dispense)

### ⚠️ RPC 템플릿 (정확히 미러)

**`dispense_prescription`** = `perform_examination`(0015:194-219) 미러:
```sql
create or replace function public.dispense_prescription(p_prescription_id uuid)
returns public.prescriptions
language plpgsql security definer set search_path = public as $$
declare v_row public.prescriptions;
begin
  if not public.has_permission('prescription.dispense') then
    raise exception 'permission denied: prescription.dispense' using errcode = 'insufficient_privilege';
  end if;
  select * into v_row from public.prescriptions where id = p_prescription_id for update;
  if not found then raise exception 'prescription not found: %', p_prescription_id using errcode = 'PT404'; end if;
  if v_row.status <> 'issued' then  -- 일방향(이미 dispensed 재발급 차단)
    raise exception 'invalid prescription transition: % -> dispensed', v_row.status using errcode = 'PT409';
  end if;
  update public.prescriptions set status = 'dispensed', dispensed_at = now(), updated_at = now()
   where id = p_prescription_id returning * into v_row;  -- dispensed_by SET 없음(설계 ④)
  return v_row;
end; $$;
revoke all on function public.dispense_prescription(uuid) from public, anon, authenticated;
grant execute on function public.dispense_prescription(uuid) to authenticated, service_role;
```
**`log_prescription_document_export`** = `log_payment_document_export`(0049:75-124) 미러하되: 게이트 `prescription.dispense`·**finalized 게이트 제거**·존재 검사 `prescriptions`(PT404)·INSERT `target_table='prescriptions'`·EXECUTE service_role only.
