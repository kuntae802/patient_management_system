## 2. 잘된 점 (강점) — _Project Lead: "다 좋았어"(4개 강점 전수 인정)_

- **🟢 0015 오더 엔진의 재사용 복리** — 5.1이 오더 생명주기 엔진(유형별 상태머신·전이 트리거·전진 RPC·`PT409`/`PT404`·`_map_pg_sqlstate`)을 한 번에 완비하자, **5.2·5.3·5.4·5.9가 신규 마이그 0건**으로 소비만 했다. "신규 마이그/권한/admin grant/SQLSTATE/감사 마스킹 = 전부 0" 설계 주장이 5.2~5.4에서 반복 검증됨. 도메인 엔진 1스토리 선투자가 후속 4스토리를 떠받쳤다.
- **🟢 DB-owns-invariants 일관 유지** — 상태머신 전이(0015 `PT409`)·재수행 차단(FR-093 소스상태 precondition)·수가 자동발생(0021 `AFTER UPDATE OF status`+`WHEN`+SECURITY DEFINER 적재)·멱등(`unique(source_type,source_id)`) 전부 Postgres 강제. FastAPI는 오케스트레이션·권한 1차선만. 10스토리 내내 흔들림 0. Epic 4 자세의 성숙.
- **🟢 적대적 리뷰의 실증 가치** — Edge Case Hunter가 실 DB(롤백 txn)로 결함을 **실증**: 5.10에서 nurse가 SECURITY DEFINER `insert_fee_item`을 직접 호출해 30000원 수가를 위조하는 권한 우회를 재현 → EXECUTE 회수 누락(코드베이스 일관 패턴 0005/0012/0036에서 5.10만 빠짐)을 청산. 동시에 거짓경보를 대량 기각(5.9 Blind 최대 우려군=원자성·auth.uid()·TOCTOU가 `authenticated_conn` 검증으로 전부 FP). 진짜 버그를 가르고 노이즈를 거른다.
- **🟢 공유 인프라 레버리지** — `_map_pg_sqlstate`(4.2)·`authenticated_conn`(단일 txn + JWT claims 주입)·`MasterSearchPicker`(2.3·4.7 ariaInvalid props, 5.2 약품·5.5 수가 재사용)·감사 마스킹 `_SENSITIVE_KEY`(notes/content/SOAP/findings 일관 커버)를 Epic 4→Epic 5 전반에서 재사용. 핸드오프 청결.

## 9. 핵심 교훈 (Key Takeaways)

1. **도메인 엔진을 1스토리에 선투자하면 후속 스토리가 마이그 0건으로 떠받쳐진다(복리).** 0015 오더 엔진이 5.2~5.9를 소비-only로 만들었다 — "전부 0" 설계 주장의 반복 검증.
2. **적대적 리뷰는 실증으로 진짜 결함을 가르고, 검증으로 거짓경보를 거른다.** Edge Hunter가 실 DB로 권한 우회를 재현(5.10)했고, `authenticated_conn` 검증으로 최대 우려군을 FP로 기각(5.9). 둘 다 가치 — 단, 자기리뷰(5.5)는 다양성을 잃으니 다른 LLM 권장.
3. **이월은 만기(소비처 도래) 시점에만 청산한다.** 마스터 불변식은 Epic 7이 소비처라 7.1 흡수, 동시성·soft-delete는 DB 최종선이 막아 이월 지속 — 일괄 청소는 소비처 없는 코드를 낳는다(2-6·3-6 정석 계승).
4. **DB-owns-invariants가 동시성 하드닝을 앱에서 면제한다.** 재수행 차단·수가 멱등·전이 무결성을 Postgres에 두면 앱-레벨 잠금 재구현이 불필요(Epic 4 결정의 Epic 5 성숙).
5. **계획 문서 < glossary/현행 코드.** 마이그 번호(0015·0017·0021)·웹 스택·병렬 worktree 번호 블록은 glossary가 단일 진실 — Epic 2~5 일관.
