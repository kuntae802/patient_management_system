## Story

As a 원무 직원,
I want finalized 수납 건에 대해 표준 「진료비 계산서·영수증」을 미리보기하고 브라우저 인쇄(`Ctrl P`)/PDF로 출력하기를,
so that 환자가 「국민건강보험법 시행규칙」 별지 서식에 맞는 표준 영수증을 받고(대분류 항목·급여/비급여·본인부담총액·기납부·납부할금액 3행 합계), 인쇄/내보내기는 감사 이벤트로 남으며 주민번호·파일명 PII가 누출되지 않는다.

> **Epic 7 다섯 번째 스토리 — 표준 문서 출력 레이어(Epic 7 "가장 어려운 도메인"의 산출물 표면).** 7.1(`0045`)~7.4(`0048`)가 수납 건을 **집계→산정→finalize(결제·내원완료·영수증번호 부여)**까지 채웠다. **7.5는 그 finalized 수납 건을 법정 서식 문서로 렌더한다**: ① `0049` `clinic_profile`(요양기관 정보 1행 마스터·seed) + `log_payment_document_export` RPC(내보내기 'read' 감사·`reveal_contact` 선례), ② FastAPI `GET .../payment/receipt`(병원·환자 masked RRN·진료과/담당의/진료기간·항목별 금액·납부·발급 정보를 한 번에 조립한 `ReceiptDocument`) + `POST .../payment/receipt/export`(감사 기록), ③ web **인쇄 전용 React 컴포넌트**(Batang serif 법정 서식 + `@media print`)가 항목별 금액표(대분류 × 급여 본인/공단·비급여 + 합계)·납부 3행을 렌더하고 `window.print()`로 인쇄/PDF 저장, **`beforeprint` 리스너가 인쇄/내보내기를 감사**한다. 7.4 finalized 패널의 "문서 출력 (준비 중)" placeholder(billing-detail.tsx L334-341)를 실제 출력으로 대체한다. **신규 라이브러리 0**(브라우저 인쇄·서버측 PDF 미사용). **주민번호 = masked only**(full reveal 이월).

**AC6 — 항목별 금액표 대분류(category) 집계(FR-113·도메인 리서치 대분류):**
- **클라 집계 함수**(receipt-document.tsx·기존 `groupByCategory`(billing-detail.tsx L90-99) 패턴 계승하되 **금액 합산**): `details` 를 `category`(진찰료/검사료/영상료/처치료/주사료/… — seed.sql L447-465 실제 값)별로 묶어 행마다:
  - **급여 본인부담금** = Σ `copay_amount_krw` where `coverage_type='covered'`
  - **급여 공단부담금** = Σ `insurer_amount_krw` where `coverage_type='covered'`
  - **비급여** = Σ `amount_krw` where `coverage_type='non_covered'`(비급여는 전액 본인부담 — 7.3: non_covered → copay=amount·insurer=0)
  - **금액 합계** = Σ `amount_krw`(전 라인)

### ⚠️ 항목별 금액표 집계 (정확히 — FR-113 표준 구조)

도메인 리서치 라인 13(대분류) + 라인 44(영수증 = 대분류 × 급여 본인/공단·비급여 + 합계 3행). 라인(`payment_details`)을 `category` 별로 그룹:
```
급여 본인부담금 = Σ copay_amount_krw  (coverage_type='covered')
급여 공단부담금 = Σ insurer_amount_krw (coverage_type='covered')
비급여        = Σ amount_krw         (coverage_type='non_covered')  -- 비급여=전액 본인부담(7.3: copay=amount·insurer=0)
금액 합계     = Σ amount_krw         (전 라인)
```
- 소계(tfoot) = 전 category 합 → **헤더 정합 불변식**: Σ category[급여본인+비급여] = `copay_amount_krw`·Σ[급여공단] = `insurer_amount_krw`·Σ[합계] = `total_amount_krw`(테스트로 단언).
- 본인부담 총액(납부할 금액) = `copay_amount_krw`. 3행 합계(FR-113) = 본인부담총액(`copay`) · 기납부(`paid_amount_krw`) · 납부할금액(`due_amount_krw`=copay-paid·7.4 전액정산이면 paid=copay→due=0).
- category 실제 값(seed.sql L447-465): 진찰료·검사료·영상료·처치료·주사료. null→"기타". 적재 순서 보존(진찰료 먼저·groupByCategory L90-99 패턴).
