# Story 5.4: 처치 오더

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 의사,
I want 처치를 오더하면 간호 워크리스트로 전달되기를,
so that 처치 수행이 추적된다.

## Acceptance Criteria

(epics.md Epic 5 / Story 5.4 — FR-070)

1. **(FR-070) 처치 오더 생성** — 진료 허브 오더 패널에서 EDI 행위 마스터 검색 피커로 처치를 오더하면, `treatment_orders` 행이 **`status='ordered'`(지시)** 로 생성되고 **지시 의사(`ordered_by`)** 가 기록된다. 처치 행위는 마스터 FK(`fee_schedule_id`)로만 — free-text 차단. 한 행위 = 한 오더(1 POST = 1 treatment_order, `prescriptions`처럼 헤더/상세 1:N 아님).
2. **(FR-070) 간호 워크리스트 라우팅** — 처치 오더는 **단일 목적지(간호)** 로 전달된다(검사·영상의 `exam_type` 같은 분류 분기 **없음** — 처치는 분기 없이 간호 직역으로만 흐름). 5.4의 라우팅 책임 = 오더가 **`status='ordered'`로 생성되어 간호(`order.read`)가 조회 가능**한 토대까지. 실제 간호 처치 워크리스트 UI·수행(perform)·재수행 차단·일상 간호기록은 **소비 스토리 5.7** 소유(아래 §스코프 경계).
3. **조회·추적성** — 한 내원의 오더된 처치 목록을 조회하면 행위명(`fee_schedules` 조인 `code`·`name`·`amount_krw`·`category`)·상태 배지(지시)·지시 시각이 표시된다. 모든 오더 생성은 감사 트리거로 기록된다(FR-080 추적성).
4. **권한·경계** — 처치 오더 = `treatment.order`(의사), 조회 = `order.read`(의사·간호·방사선). 권한 미보유(원무 = 오더 baseline) → 403. 생성은 **service_role 직접 INSERT**(register/RPC 아님 — 자유 CRUD), 권한은 쓰기 직전 동일 트랜잭션 재평가(TOCTOU).

### 처치 라우팅 (FR-070) — 단일 목적지(간호), 분류 축 없음 [Source: 0015:101, epics.md:945-955]
