# Story 5.3: 검사 · 영상 오더

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 의사,
I want 진단검사·영상검사를 오더하면 적절한 직역 워크리스트로 전달되기를,
so that 수행 주체가 명확히 분기된다.

## Acceptance Criteria

(epics.md Epic 5 / Story 5.3 — FR-060·FR-061)

1. **(FR-060) 검사·영상 오더 생성** — 진료 허브 오더 패널에서 EDI 행위 마스터 검색 피커로 검사/영상을 오더하면, `examinations` 행이 **`status='ordered'`(지시)** 로 생성되고 **지시 의사(`ordered_by`)** 가 기록된다. 검사 종류(행위)는 마스터 FK(`fee_schedule_id`)로만 — free-text 차단. 한 행위 = 한 오더(1 POST = 1 examination, `prescriptions`처럼 헤더/상세 1:N 아님).
2. **(FR-061) 유형별 워크리스트 라우팅 분기** — 오더 생성 시 **`exam_type`(`lab` 진단검사 / `imaging` 영상검사)** 을 명시 지정한다. `exam_type`이 워크리스트 라우팅의 분류 축 — 영상검사(`imaging`)는 방사선사 워크리스트(5.8), 진단검사 검체(`lab`)는 간호 워크리스트(채취 후 외부 의뢰, 다운스트림)로 분기되는 **토대**를 제공한다. 5.3의 책임은 오더가 **올바른 `exam_type`으로 생성·조회**되는 것까지 — 워크리스트 UI·수행·외부 의뢰 결과 기록은 소비 스토리(5.7/5.8/5.9) 소유(아래 §스코프 경계).
3. **조회·추적성** — 한 내원의 오더된 검사·영상 목록을 조회하면 행위명(`fee_schedules` 조인 `code`·`name`·`amount_krw`·`category`)·`exam_type`(검사/영상 라벨)·상태 배지(지시)·지시 시각이 표시된다. 모든 오더 생성은 감사 트리거로 기록된다(FR-080 추적성).
4. **권한·경계** — 검사·영상 오더 = `examination.order`(의사), 조회 = `order.read`(의사·간호·방사선). 권한 미보유(원무 = 오더 baseline) → 403. 생성은 **service_role 직접 INSERT**(register/RPC 아님 — 자유 CRUD), 권한은 쓰기 직전 동일 트랜잭션 재평가(TOCTOU).

### exam_type 라우팅 (FR-061) — 분류 축은 의사가 명시 지정 [Source: 0015:81, 99, seed.sql:300-318, masters.ts:243-288]

- **`exam_type`(`lab`/`imaging`)이 워크리스트 라우팅의 유일한 강제 분류 축**(CHECK enum). 영상(`imaging`)→방사선사 워크리스트(5.8), 검체(`lab`)→간호 워크리스트(채취 후 외부 의뢰, 다운스트림). `(exam_type, status)` 부분 인덱스(0015:99)가 워크리스트 조회의 토대.
- **⚠️ `fee_schedules.category`로 추론 금지** — `category`는 자유 텍스트(CHECK 없음, "검사료"/"영상료"/"처치료" 등 시드 라벨). "검사료"엔 심전도·내시경 등 비영상 검사가 섞이고 강제 분류가 아니다. **`exam_type`은 의사가 UI 세그먼트 토글로 명시 선택**(더 제한적 옵션 — project-context §모호 시 제한적). category는 표시·참고용(pay-chip 분류명).
- **5.3의 라우팅 책임 = 오더가 올바른 `exam_type`으로 생성·조회됨까지.** 실제 워크리스트 UI·소비·라우팅 분기 렌더는 소비 스토리(5.8 방사선·다운스트림 간호) 소유. 5.3이 워크리스트를 만들지 않는다(아래 §스코프 경계).
- 데모 행위(seed.sql:300-318): lab 후보 = `C3800`(CBC)·`C5400`(요검사)·`D2700`(HbA1c)·`E6541`(심전도)·`F6310`(알레르기)·`HB010`(비내시경); imaging 후보 = `HA201`(흉부촬영)·`HA401`(복부촬영). 테스트·데모는 이 코드 사용.
