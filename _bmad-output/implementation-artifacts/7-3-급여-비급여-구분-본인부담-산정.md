## Story

As a 원무 직원,
I want 수납 상세 라인의 급여/비급여를 환자의 보험유형에 따라 본인부담금·공단부담금으로 산정하기를,
so that 환자에게 청구할 정확한 본인부담금이 자동으로 산정·표시되고, 7.4(finalize·결제)와 7.5~7.6(영수증·세부산정내역서)가 이 산정 결과 위에서 동작한다.

> **Epic 7 세 번째 스토리 — 산정(pricing) 레이어.** 7.1(`0045`)이 copay 컬럼을 선언하고, 7.2(`0046` `build_payment`)가 `payment_details` 라인을 적재·`total/covered/non_covered` 헤더 롤업까지 채웠다. **7.3은 그 라인에 본인부담을 산정**한다: `copay_policies` 정책 테이블(보험유형×급여구분→본인부담률)을 도입하고, `price_payment` DB 함수가 라인별 `copay_rate`·`copay_amount_krw`·`insurer_amount_krw`를 채우며 헤더 `copay_amount_krw`·`insurer_amount_krw`를 롤업한다. 산정은 **집계에 이어 자동 실행**(`POST .../payment`가 `build_payment`→`price_payment`를 한 트랜잭션으로 호출).

**AC1 — 본인부담률 정책 테이블 `copay_policies` + 시드(0047·마이그 임베드 — 운영 필수 참조 데이터):**
`0047`에서 `copay_policies` 테이블이 생성된다(보험유형×급여구분 → 본인부담률 단일 진실):
- 컬럼: `id uuid pk default gen_random_uuid()` · `insurance_type text not null check (in ('health_insurance','medical_aid','auto_insurance','self_pay'))` · `coverage_type text not null check (in ('covered','non_covered'))` · `copay_rate numeric(4,3) not null check (copay_rate between 0 and 1)` · `created_at`/`updated_at timestamptz not null default now()` · `unique (insurance_type, coverage_type)`.
- **시드 8행(마이그 임베드·`insert ... on conflict (insurance_type, coverage_type) do nothing` — permissions 부트 패턴, ⚠️ `seed.sql` 아님)**: price_payment 정확성이 전 환경(운영 포함)에서 의존하므로 dev 전용 `seed.sql`이 아니라 **마이그레이션에 임베드**한다(permissions/role_permissions admin 부트 grant와 동일 posture).
  - covered: `health_insurance`/`covered`/`0.300` · `medical_aid`/`covered`/`0.150` · `auto_insurance`/`covered`/`0.000` · `self_pay`/`covered`/`1.000`.
  - non_covered(전 유형 100% — 보험유형 무관 환자 전액): `health_insurance`/`non_covered`/`1.000` · `medical_aid`/`non_covered`/`1.000` · `auto_insurance`/`non_covered`/`1.000` · `self_pay`/`non_covered`/`1.000`.
- 권한 posture(비민감 참조 데이터 — 마스터 0006/0007 미러): `revoke all on public.copay_policies from anon` · `grant select to authenticated, service_role`(쓰기 없음 — 마이그/시드 소유). **AC2 — 본인부담 산정 함수 `price_payment(encounter_id)`(0047·멱등·산정 로직 = DB 소유):**
`0047`에서 `price_payment(p_encounter_id uuid) returns uuid`(`SECURITY DEFINER`·`set search_path=public`·`language plpgsql`)가 생성된다. 한 내원의 draft 수납 건 라인에 본인부담을 산정한다:
- **헤더+보험유형 룩업(1회)**: `select pay.id, pay.status, pat.insurance_type into ... from public.payments pay join public.encounters e on e.id = pay.encounter_id join public.patients pat on pat.id = e.patient_id where pay.encounter_id = p_encounter_id`. 헤더 없음(빌드 전) → **`return null`**(산정 대상 없음 — build_payment가 선행 생성).
- **상태 가드**: `status≠'draft'`(finalized/cancelled)면 **산정 변경 없이** payment_id만 반환(finalize 후 본인부담 동결 — 7.4 결정 보존, build_payment 동형).
- **라인별 산정(정책 JOIN·멱등·change-guard)**: 각 `payment_detail`에 대해 정책 룩업 `copay_policies` `(v_insurance_type, pd.coverage_type)` → `v_rate = coalesce(pol.copay_rate, 1.0)`(정책 미존재 = 보수적 100% 환자부담·미청구 방지). 산정:
  - `copay = case when v_rate >= 1 then pd.amount_krw  when v_rate <= 0 then 0  else (floor(pd.amount_krw * v_rate / 10) * 10)::int end`  (⚠️ **10원 미만 절사는 분할 케이스(0<rate<1)에만** — rate=1.0이면 copay=amount 전액·rate=0이면 copay=0, 절사로 인한 1원 누락 차단).
  - `insurer = pd.amount_krw - copay`  (차액 흡수 → **라인 불변식 `amount = copay + insurer` 항상 성립**).
  - `copay_rate = v_rate` 스냅샷(적용 본인부담률 고정).
  - **변경된 라인만 UPDATE**(`where (pd.copay_rate, pd.copay_amount_krw, pd.insurer_amount_krw) is distinct from (target...)` — 재진입·미변경 라인은 트리거 미발화·감사 노이즈 0·build_payment 헤더 롤업 change-guard 동형).
- **헤더 롤업(change-guard)**: `update payments set copay_amount_krw = Σ copay, insurer_amount_krw = Σ insurer, updated_at = now() where id = v_payment_id and (copay_amount_krw, insurer_amount_krw) is distinct from (Σcopay, Σinsurer)`. 결과 불변식: **`total_amount_krw = copay_amount_krw + insurer_amount_krw`**(전 라인 amount=copay+insurer의 합).
- 반환: payment_id.
- 함수 EXECUTE posture(0046 build_payment 동형): `revoke all on function public.price_payment(uuid) from public, anon, authenticated` + `grant execute on function public.price_payment(uuid) to service_role`(service_role/FastAPI만 — authenticated 직접 호출 차단·산정 위조 방지).

**AC3 — `POST /v1/encounters/{encounter_id}/payment` = build→price 원자(진입 시 자동 집계+산정·멱등):**
기존 빌드 엔드포인트(7.2·새 엔드포인트 추가 없음)가 **집계에 이어 산정까지** 수행한다. `api/app/core/db.py`의 `build_payment` `_op`(동일 트랜잭션) 안에서: `_require_payment_manage`(산정도 쓰기 — 기존 게이트 충분) → 내원 존재 검사 → `select public.build_payment($1)`(집계) → **`select public.price_payment($1)`(산정·신규)** → 헤더+라인 재조회(copay/insurer 채워진 상태). 두 RPC가 한 `authenticated_conn` 트랜잭션 = 원자성(NFR-041). 멱등(재호출 = 신규 fee_item만 집계 + 전 라인 재산정·미변경시 no-op). 미존재 내원 → 404, 권한 미보유 → 403.

### ⚠️ 산정 공식 (정확히 — 라인 불변식 보존)

라인별(`pd.amount_krw` = quantity×unit·7.2 스냅샷):
```
v_rate = coalesce((select copay_rate from copay_policies
                   where insurance_type = v_insurance_type
                     and coverage_type = pd.coverage_type), 1.0)   -- 정책 미존재 = 100% 환자(보수적)
copay  = case when v_rate >= 1 then pd.amount_krw                  -- 전액 본인(self_pay·비급여)
              when v_rate <= 0 then 0                              -- 전액 공단(자보)
              else (floor(pd.amount_krw * v_rate / 10) * 10)::int  -- 10원 절사 본인부담
         end
insurer = pd.amount_krw - copay                                   -- 차액 흡수 → amount=copay+insurer 항상 성립
copay_rate(스냅샷) = v_rate
```
헤더: `copay_amount_krw = Σ copay` · `insurer_amount_krw = Σ insurer` → `total_amount_krw = copay + insurer`. (비급여 insurer=0 → insurer는 covered 라인에서만 발생. 그러나 헤더 항등은 total=copay+insurer로 단순.)

⚠️ **절사 가드 필수**: rate=1.0인 self_pay/비급여 라인을 `floor(amount*1.0/10)*10`으로 계산하면 amount가 10의 배수가 아닐 때 1원 누락 + insurer에 잔돈 발생(self_pay인데 공단부담 발생 = 오류). 그래서 `rate>=1 then amount`·`rate<=0 then 0` 분기를 둔다.
