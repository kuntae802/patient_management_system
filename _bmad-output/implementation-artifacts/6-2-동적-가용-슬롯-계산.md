# Story 6.2: 동적 가용 슬롯 계산

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a **환자·원무**,
I want **실제로 예약 가능한 슬롯만 보기를**,
so that **잘못된 시간(휴진·마감·근무외)에 예약하지 않는다.**

## Acceptance Criteria

**[AC1] 동적 슬롯 산출 = 근무 − 휴진 − 기예약 (FR-012)**
- **Given** 의사 근무표(`doctor_schedules`)·휴진(`doctor_time_offs`)·기존 예약(`appointments`)이 있을 때
- **When** 특정 의사·날짜의 가용 슬롯을 계산하면
- **Then** `근무 블록 − 휴진 구간 − 활성 예약(status='booked')`으로 산출된 **가능 슬롯만** `available` 로 노출되고, 차감된 구간은 사유별 비활성 상태로 표시된다.

**[AC2] 슬롯 그리드 비활성 렌더 (음영 비의존)**
- **Given** 휴진일·마감(예약됨) 슬롯·지난 슬롯에 대해
- **When** 슬롯 그리드를 렌더하면
- **Then** 각 슬롯이 `available`(선택가능) / `booked`(마감) / `time_off`(휴진) / `past`(지남) 중 하나로 **채움+테두리+라벨**(음영 비의존, UX-DR 색·명도 원칙)로 구분되고, 비활성 슬롯은 선택 불가하다.

**[AC3] 비활성/미배정 의사 = 빈 가용**
- **Given** 휴직·퇴사(비-active) 의사 또는 해당 요일에 근무표가 없는 의사에 대해
- **When** 슬롯을 계산하면
- **Then** 가용 슬롯이 0건으로 반환된다(404 아님 — 정상 빈 결과). (6.1 이월: "스케줄 employment 재검증 → 6.2 슬롯계산이 active 의사 조인 필터로 흡수")

**[AC4] 권한 게이트 (appointment.read)**
- **Given** 신규 권한 `appointment.read` 미보유 계정(예: nurse)으로
- **When** 슬롯 조회 엔드포인트를 호출하면
- **Then** 403(forbidden)이 반환된다. 보유 계정(reception·admin)은 200.

**[AC5] appointments 스키마·불변식 = DB 소유 (6.3 더블부킹 토대)**
- **Given** `appointments` 테이블이 생성될 때
- **When** 같은 의사·겹치는 시간의 활성(`status='booked'`) 예약을 2건 넣으면
- **Then** DB EXCLUDE 제약(`23P01`)이 두 번째를 거부한다(6.3 예약 쓰기가 이 제약을 409 로 표면화 — 본 스토리는 제약 토대 + seed 직접 INSERT 검증까지).

> **범위 노트(에픽 6.2):** "동적 가용 슬롯 계산"이 본 스토리의 본질이며, **AC1·AC2·AC3·AC4** 가 시연 가능한 결과물이다. 단, AC1 의 "기예약" 차감을 만족하려면 `appointments`(예약 본체) 테이블이 존재해야 하므로 — 6.1 이 6.2 로 명시 이월한 **"appointments 소유 결정"** 에 따라(Option A 확정) — 본 스토리가 `appointments` 풀 스키마를 함께 생성한다(**AC5**). 예약 **쓰기**(생성/변경/취소 RPC·전이 트리거)·캘린더·booking-peek·더블부킹 409 인라인 표면화는 **6.3/6.4** 소유.
