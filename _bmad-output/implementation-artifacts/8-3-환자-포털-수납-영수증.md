# Story 8.3: 환자 포털 — 수납 · 영수증

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a **환자**,
I want **본인의 수납 내역과 영수증을 쉬운 말로 보기를**,
so that **내가 낸 진료비와 결제 내역을 스스로 확인한다**.

> **Epic 8 세 번째 스토리 — 환자 포털 '마이' 탭을 채운다.** 8.1(내 기록 카드)·8.2(카드 펼침 처방·검사)가 "내 기록" 탭을 완성했고, self-read 패턴(`/patients/me/*` + `get_current_patient` + `where p.auth_uid=$1` + 소유검증→404)을 확립했다. 8.3은 그 패턴을 **수납 도메인**에 한 번 더 적용한다: portal 페이지("영수증 조회는 추후 제공됩니다" placeholder)를 **마이 탭 = 내 진료비·영수증 화면**으로 채우고, 7.1~7.5가 쌓은 **finalized 수납 건**을 환자 언어로 비춘다. **읽기 전용 · 세션 uid 스코프(본인 외 0건) · 새 데이터 생성 0 · 마이그레이션 0**(금액·카테고리·요양기관 정보가 이미 다 존재 — 쉬운말 큐레이션 컬럼 불요).

## Acceptance Criteria

> 출처: `epics.md` Story 8.3(L1389-1401) — 환자 세션에서 수납 내역을 열면 → 본인 수납 내역·영수증이 조회된다(FR-122). **And** 12시간 표기·쉬운 말·본인 데이터 스코프(RLS)가 일관 적용된다.
> ⚠️ **설계 결정 3건(사용자 확정 2026-06-24, AskUserQuestion)** — Dev Notes §경계·설계 결정 참조. ① **화면 위치 = '마이' 탭(`/portal`) 전용 수납 내역 화면**(placeholder 대체·mockup IA 마이). ② **영수증 = 환자 친화 요약(화면) + "영수증 인쇄·저장"으로 7.5 법정 서식(`receipt-document.tsx`) 재사용**(화면은 쉬운 말, 인쇄는 공식 영수증). ③ **조회 범위 = finalized(결제완료)만**(draft·cancelled 제외).
> 마이그레이션 = **없음**(7.6 선례 — 기존 인프라 전부 재사용). 신규 식별자(스키마·함수·엔드포인트·웹 컴포넌트)는 glossary 등재(AC8).

**AC1 — '마이' 탭 = 수납 내역 리스트 (FR-122, UX-DR17):**
**Given** 환자 세션(uid 스코프)에서
**When** '마이' 탭(`/portal`)을 열면
**Then** 본인의 **finalized 수납 건**이 **최근순** 카드로 표시된다 — 각 카드에 **날짜(12h·KST) · 요양기관/진료과 · 납부 금액(`paid_amount_krw`·"원") · 결제 완료 배지(색 비의존 글리프+라벨)** 가 쉬운 말로 표시되고, 카드를 탭하면 영수증 상세로 이동한다(≥44px 터치 타깃, 셰브런 `›`). 상단에 **신뢰 노트**(8.1)가 상시 표시된다. 마이 탭의 계정 동작(본인 진료기록 연결·로그아웃)은 유지된다.

**AC2 — 영수증 상세 = 환자 친화 요약 (FR-122·UX-DR23 쉬운 말):**
**Given** 마이 탭에서 수납 카드를 탭해 영수증 상세(`/receipts/{encounter_id}`)를 열면
**When** 영수증을 렌더하면
**Then** **쉬운 말 요약**이 큰 타입·12h·KRW 정수로 표시된다 — 요양기관·진료과·담당의·진료일 / **항목 대분류별 금액**(진찰료·검사료 등·category 그대로·"기타" 폴백) / **총 진료비(`total_amount_krw`) · 건강보험에서 낸 금액(`insurer_amount_krw`) · 내가 낸 금액(`paid_amount_krw`)** / 결제수단(`PAYMENT_METHOD_LABEL`). 금액은 전부 DB 산정값(클라 산술 금지·표시 그룹 합산만). 화면에는 **법정 serif 서식을 쓰지 않는다**(DESIGN: legal-serif=화면 UI 미사용).

**AC3 — 영수증 인쇄·저장 = 7.5 법정 서식 재사용 (FR-113 재사용·UX-DR14):**
**Given** 영수증 상세 화면에서
**When** "영수증 인쇄·저장" 버튼을 누르면
**Then** 7.5 `ReceiptDocument`(Batang serif 「국민건강보험법 시행규칙」 별지 서식·항목별 금액표·납부 3행)가 **인쇄 전용으로 출력**된다 — 화면에는 숨기고(`hidden print:block`) `@media print`(`.receipt-paper`)로만 렌더, `window.print()` 호출. 동일 데이터(`ReceiptResponse`)를 친화 요약(화면)과 법정 서식(인쇄)이 공유한다. **인쇄 파일명/문서 제목에 PII 금지**(`document.title`=`영수증_{chart_no}`·불투명·이름/주민번호 금지).

**AC4 — self-scope 경계 · finalized 게이트 · 빈/예외 상태:**
**Given** RLS·세션 uid 경계가 강제되는 상태에서
**When** 수납 내역/영수증을 받으면
**Then** **본인 수납 건만** 반환된다 — 리스트는 `where p.auth_uid=$1 and pay.status='finalized'`, 영수증 상세는 **소유 검증**(`encounter_id`의 payment가 본인 것이고 finalized 인지) 미일치 시 **404**(존재/비소유/비-finalized 구분 노출 금지·IDOR 차단). 수납 건이 **0건**이면 빈 상태 안내("아직 결제 내역이 없어요")를, **미연결 환자**(self 레코드 없음)면 온보딩으로 유도한다(8.1 패턴). 시간은 12시간(오후 2:30)·KST·쉬운 말이 일관 적용된다.

**AC5 — PII·임상 데이터 경계 (project-context·UX-DR22):**
**Given** 환자에게 수납 데이터를 투영할 때
**When** 응답·렌더하면
**Then** **raw 주민번호·연락처·암호화/해시 컬럼·내부 `*_by` id 는 절대 미투영**한다(친화 요약은 RRN 미표시·법정 인쇄 재사용분은 7.5 `resident_no_masked`만·본인 데이터). 금액은 KRW 정수("원")·`tabular-nums`. 라우트는 `encounter_id`(불투명 UUID). **환자 자기 데이터 조회는 감사 미적재**(8.1·8.2 self-read 선례 — UX-DR22 내보내기 감사는 직원이 타인 민감 문서를 다룰 때의 정책이며, 환자 본인 영수증 열람·인쇄는 reveal 아님·감사 미발생). [↳ deferred: 환자 본인 내보내기 감사가 필요하면 후속.]

### 🔒 보안 경계 — service_role 현실 + self-scope (가장 중요)
**FastAPI는 service_role(로컬 postgres) 연결 → RLS 우회**(`api/app/core/db.py:3` 주석). `_run_authed(sub)` 는 `auth.uid()` 만 주입할 뿐 RLS 필터를 적용하지 않는다.
- ⇒ **리스트**: `where p.auth_uid=$1`(=sub)가 경계선(`fetch_self_encounters` db.py:1402 선례). `pay.status='finalized'` 필터로 draft/cancelled 차단.
- ⇒ **영수증 상세**: **소유 검증** `and pat.auth_uid=$2`(요청 encounter_id 의 payment 가 본인 것인지). 미일치/비-finalized → **None 반환 → 404**(존재/비소유/비-finalized 구분 노출 금지·IDOR 차단·8.2 `fetch_self_encounter_detail` db.py:1452 소유검증 선례).
- ⇒ **7.5 직원 receipt(409 비-finalized)와 self 차이**: 직원은 finalized 패널 진입이라 비-finalized=409(invalid_transition)가 맞지만, **환자 self 는 draft 존재 자체를 노출하지 않아야** 하므로 비-finalized=404(소유 미일치와 동일 응답). 의도된 이탈.
- **안티패턴 금지**: 클라이언트 Supabase 직접조회로 payments/payment_details 를 읽지 말 것(컬럼 투영 통제 불가·요양기관/진료과 조인이 RLS에 막힘). 환자 수납 데이터의 투영은 FastAPI가 소유(8.1/8.2 동일).
- **감사**: 환자 본인 영수증 열람·인쇄는 **감사 미적재**(8.1/8.2 self-read 무감사 선례).
