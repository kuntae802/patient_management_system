# Story 5.7: 간호 처치 수행 · 재수행 차단 · 일상 간호기록

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 간호사,
I want 처치 워크리스트에서 지시된 처치 오더를 수행 처리(처치기록 내용 첨부)하되 이미 수행된 건은 막히고, 오더 없이도 일상 간호기록을 남기기를,
so that 처치 중복·누락이 0이 되고 간호 활동이 추적 가능하게 기록된다.

## Acceptance Criteria

1. **(FR-090·FR-092) 처치 워크리스트 조회 + 처치 오더 수행·처치기록** — 간호사는 `/nurse/treatments`에서 **오늘(KST) 활성 내원**(`registered`·`in_progress`) 중 **미수행 처치 오더(`status='ordered'`)가 있는 내원** 목록을 보고(환자명·차트번호·진료과·미수행 처치 건수·가장 오래된 미수행 지시시각), 한 내원을 선택해 그 내원의 지시된 처치 오더를 조회·**수행 처리**한다. 수행하면 `perform_treatment_order` RPC가 오더를 `ordered → performed`로 전환하고 **수행자(`performed_by`)·수행시각(`performed_at`)을 자동 기록**하며, **처치기록 내용(`content`)이 입력되면** 해당 오더에 연결된 간호기록(`nursing_record`·`treatment_order_id` 부착)이 남는다. 게이트 = `treatment.perform`(간호 직무). 모든 전이·기록이 감사 로그에 남는다.

2. **(FR-093·UX-DR21 ⑤) 재수행 차단(중복 방지)** — 이미 수행된(`performed`) 오더에 대해 재수행을 시도하면 **mutation 중 버튼 disable(1차선)** + **수행 완료 오더 = 잠금 표시(수행 버튼 미노출·"수행 완료"·수행자/시각 표시, UX-DR21 ⑤ "already-done state")**로 UI가 막고, stale 탭 등으로 우회 호출 시 **상태머신(최종선)**이 `409 invalid_transition`으로 차단한다(`perform_treatment_order` RPC의 소스상태 precondition `status<>'ordered'`). 웹은 409를 토스트 + 워크리스트/오더 재로드로 우아하게 처리한다. **임상 해석(중요)**: "재수행 차단"은 *동일 오더*의 재수행만 막는 것 — 동일 처치를 임상적으로 반복해야 하면 의사가 **신규 처치 오더를 발행**한다(FR-093 vs FR-041 재기록 허용 철학 충돌은 "같은 오더 두 번 불가 / 새 오더는 가능"으로 해소).

3. **(FR-094) 오더 없는 일상 간호기록** — 간호사는 처치 오더 연결 없이도(`treatment_order_id` NULL) 선택한 내원에 **일상 간호기록(`content`)을 남길 수 있다**. 게이트 = `nursing.record`(신규 권한). 기록자(`recorded_by`)·기록시각(`recorded_at`)이 자동 부착되고 감사 로그가 남는다. 빈 내용은 3중 방어(클라 가드 → 서버 Pydantic `min_length` 422 → DB CHECK)로 차단된다.
